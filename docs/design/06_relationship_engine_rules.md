# 关系引擎规则 v0.1

本文件把 v0.1.0 的关系概念转成可计算、可测试、可开发的纸面规则。这里仍然只写规则，不写代码。

## 1. 设计目标

关系引擎负责回答一个问题：一次事件中的玩家选择和 NPC 真实行为，如何影响真实值、感知反馈、破绽、记忆和关系阶段。

设计目标：

- 把事件从“说明文”推进为可结算的规则。
- 保留真实值与感知值偏差，不让玩家直接看到真实数值。
- 让玩家和 NPC 的行为都能影响关系。
- 用行为标签和上下文推导后果，避免把事件写成固定数值表。
- 让同一行为在不同性格、压力、记忆和阶段下产生不同结果。

## 2. 输入数据

关系引擎至少需要以下输入：

| 输入 | 说明 | 主要用途 |
| --- | --- | --- |
| 玩家角色档案 | 基础信息、时间信息、性格维度、价值倾向 | 判断玩家行为倾向和感知偏差 |
| NPC 角色档案 | 与玩家同结构的角色档案 | 判断 NPC 主动行为、反应和隐瞒策略 |
| 当前关系阶段 | 正在聊天、暧昧中、刚恋爱等 | 限制事件池和阶段变化 |
| 当前日期和时间段 | 第几天、早上/白天/晚上/深夜 | 判断日程、回复窗口、熬夜后果 |
| 双方日程 | 上班、上学、求职、休息、社交安排 | 判断真忙、失约、见面机会 |
| 当前事件 | 事件 ID、真实层、感知层、后果层 | 提供本次结算对象 |
| 玩家选择 | 玩家在感知层做出的选择 | 转成行为标签并影响 NPC 反应 |
| NPC 真实行为 | NPC 实际做了什么 | 影响真实值、破绽和后续钩子 |
| 过去记忆 | 未解决冲突、旧账、修复记录 | 改变同类事件权重 |
| 当前压力 | 双方生活压力和关系压力 | 影响反应强度、撒谎、冷处理 |
| 边界协议 | 异性社交、公开关系、隐私、亲密节奏等规则 | 判断是否越界和是否进入记忆 |
| 真实值 | 好感、信任、安全感、压力、矛盾等 | 内部结算和阶段判定 |
| 感知值 | 玩家主观判断和模糊反馈状态 | 生成玩家可见文本 |

## 3. 行为标签

事件结果不能直接写“好感 +5”。玩家和 NPC 的行为先转成标签，再结合上下文结算。

| 行为标签 | 含义 | 通常影响哪些系统 | 哪些性格喜欢 | 哪些性格反感 | v0.1 是否启用 |
| --- | --- | --- | --- | --- | --- |
| 坦白 | 主动说明关键事实，不故意省略重点 | 信任、安全感、破绽、记忆 | 高责任感、高安全依恋、低冲突回避 | 高控制欲但期待对方服从者可能仍不满 | 启用 |
| 追问 | 对含糊信息继续询问 | 感知安全感、压力、破绽 | 高边界敏感、高查证倾向 | 高冲突回避、高自主边界 | 启用 |
| 温和确认 | 用低攻击性方式确认信息 | 信任、压力、修复余量 | 高共情、高责任感、低防御 | 高试探倾向可能觉得力度不够 | 启用 |
| 强硬质问 | 以高强度方式要求解释 | 压力、矛盾、破绽、分手危机 | 高占有欲、高控制倾向 | 高边界敏感、高冲突回避 | 启用 |
| 回避 | 不正面处理问题 | 未解决冲突、失望、压力 | 高冲突回避、低表达直接度 | 高安全感需求、高责任感 | 启用 |
| 冷处理 | 暂停回复或降低互动热度 | 安全感、未解决冲突、旧账 | 高情绪恢复慢、高惩罚倾向 | 高安全感需求、高沟通需求 | 启用 |
| 撒谎 | 主动给出与事实不符的信息 | 信任、破绽、愧疚、危机 | 高短期避险、高冲突回避 | 高诚实倾向、高边界敏感 | 启用 |
| 半真半假 | 说部分事实但省略关键对象、目的或时间 | 破绽、信任、记忆 | 高冲突回避、高被发现风险感知 | 高查证倾向、高安全感需求 | 启用 |
| 隐瞒重点 | 不主动说关键事实 | 破绽、信任、第三者线索 | 高自主边界、高回避 | 高边界敏感、高公开需求 | 启用 |
| 反向指责 | 把问题转向“你不信任我”等指责 | 压力、矛盾、未解决冲突 | 高防御、高双标倾向 | 高责任感、高修复需求 | 启用 |
| 查证 | 通过朋友圈、共同朋友、时间线等验证 | 破绽、信任、压力 | 高不安、高查证倾向 | 高隐私边界、高自主需求 | 启用 |
| 设边界 | 明确以后能做什么、不能做什么 | 边界协议、信任、阶段稳定 | 高责任感、高规则意识 | 高自由需求、高双标倾向 | 启用 |
| 道歉 | 承认自己的行为造成影响 | 修复余量、信任、失望 | 高共情、高责任感 | 高防御或高胜负心 | 启用 |
| 补偿 | 用行动修复伤害或落空期待 | 失望、关系满意度、记忆 | 高被重视需求、高责任感 | 认为补偿替代沟通的人会反感 | 启用 |
| 情绪爆发 | 情绪失控、说重话或扩大冲突 | 压力、矛盾、旧账 | 高表达直接但低稳定者 | 高面子压力、高安全需求 | 启用 |
| 公共场合争吵 | 在朋友、同事或旁观者面前冲突 | 面子压力、失望、记忆 | 高当场解决倾向 | 高社交展示欲、高面子压力 | 启用 |
| 私下沟通 | 避开公共场合后处理问题 | 修复、信任、未解决冲突 | 高共情、高面子压力、高责任感 | 高当场表达需求 | 启用 |
| 拉黑 | 切断消息渠道或临时断联 | 安全感、失望、危机、记忆 | 高惩罚倾向、高情绪化 | 高安全感需求、高沟通需求 | 启用 |
| 复联 | 拉黑、冷战或分手后重新联系 | 修复、旧账、复合可能 | 高依恋、高修复意愿 | 高边界清晰、低反复容忍 | 启用 |
| 试探 | 故意制造不确定性观察对方反应 | 安全感、考验倾向、记忆 | 高考验倾向、高不安 | 高诚实倾向、高边界敏感 | 启用 |
| 报复性行为 | 因不满故意做类似或更强行为 | 信任、矛盾、分手危机 | 高胜负心、高失望 | 高责任感、高长期导向 | 启用 |
| 接受解释 | 暂时相信对方说明 | 感知信任、冲突、破绽延迟 | 高信任、高冲突回避 | 高查证倾向、高不安 | 启用 |
| 暂时观察 | 不立刻定性，保留怀疑 | 感知反馈、后续查证、记忆 | 高理性、高不安但能克制 | 高明确性需求 | 启用 |
| 分手 | 明确结束关系或提出结束关系 | 阶段变化、记忆、复合难度 | 高原则边界、高失望 | 高依恋、高复合倾向 | 启用 |

## 4. 事件上下文

行为标签只有放进上下文才有意义。关系引擎至少判断以下上下文：

- 事件真实严重度：普通误会、轻微越界、明显越界、原则性事件。
- 事件发生时间：白天延迟回复和深夜失联的含义不同。
- 是否重复发生：第一次长时间不回和第三次解释矛盾的后果不同。
- 是否公开场合：公共场合会额外影响面子压力和记忆强度。
- 是否涉及第三者：普通社交、潜在暧昧、明确骑驴找马需要区分。
- 是否已有边界协议：先前说清过的规则被破坏，后果更重。
- 是否处于高压力日：压力可以解释冷淡，但不能消除所有后果。
- 是否存在修复余量：过去的坦白和补偿会提高修复空间，旧账会降低修复空间。

## 5. NPC 反应计算

NPC 反应强度用规则文字描述，不写代码：

```text
NPC反应强度 = 事件真实严重度
+ NPC当前压力
+ NPC边界敏感度
+ NPC情绪稳定度反向
+ 过往同类记忆
+ 玩家表达方式强度
- NPC对玩家信任
- 当前关系修复余量
```

各项解释：

- 事件真实严重度：事件越接近原则性错误，反应越强。
- NPC 当前压力：工作、上学、求职、家庭、经济、身体和关系压力越高，越难稳定回应。
- NPC 边界敏感度：越重视隐私、异性社交或公开关系边界，越容易对越界行为强反应。
- NPC 情绪稳定度反向：情绪稳定度越低，越容易爆发、冷处理或拉黑。
- 过往同类记忆：类似事件越多，旧账越容易被触发。
- 玩家表达方式强度：温和确认、追问、强硬质问、公共场合争吵的刺激强度不同。
- NPC 对玩家信任：信任越高，越愿意解释和修复。
- 当前关系修复余量：近期有效沟通、道歉、补偿越多，越容易把冲突控制在可修复范围。

反应输出不只是一种结果。NPC 可能选择解释、回避、反击、冷处理、设边界、拉黑、复联或分手。

## 6. 撒谎与隐瞒计算

沿用母版公式：

```text
撒谎倾向 = 诚实倾向 + 冲突回避 + 当前压力 + 撒谎收益 + 被发现风险 + 过往成功经验 + 愧疚感 + 关系满意度
```

| 项目 | 规则解释 | 可能结果 |
| --- | --- | --- |
| 诚实倾向 | 反向约束。诚实越高，越不倾向完全撒谎 | 坦白、半真半假、主动补充细节 |
| 冲突回避 | 越高越不想正面解释 | 模糊回答、隐瞒重点、冷处理 |
| 当前压力 | 压力越高，越倾向短期省事 | 敷衍、回避、反向指责 |
| 撒谎收益 | 隐瞒后能避免冲突、保留社交或维持形象 | 半真半假、完全撒谎 |
| 被发现风险 | 风险高可能降低撒谎，也可能促使更复杂谎言 | 坦白、精细化说法、删除线索 |
| 过往成功经验 | 过去隐瞒成功越多，越容易重复 | 隐瞒重点、继续模糊 |
| 愧疚感 | 高愧疚可能导致坦白，也可能导致补偿和防御性反击 | 道歉、补偿、反向指责 |
| 关系满意度 | 满意度低会降低坦白意愿，满意度高可能提高修复意愿 | 坦白、敷衍、分手前冷淡 |

撒谎不是二选一，而是在以下策略中选择：

- 坦白。
- 半真半假。
- 隐瞒重点。
- 模糊回答。
- 完全撒谎。
- 反向指责。

## 7. 真实值变化

真实值变化只在系统内部发生，文档阶段用方向描述：

- 小幅上升。
- 中幅上升。
- 明显上升。
- 小幅下降。
- 中幅下降。
- 明显下降。
- 写入长期记忆。
- 提高阶段变化权重。

结算原则：

- 坦白通常有利于真实信任，但如果事实严重，仍可能触发危机。
- 温和确认通常比强硬质问保留更多修复余量。
- 半真半假通常短期降低冲突，长期提高破绽和信任风险。
- 冷处理通常降低当下冲突，但提高未解决冲突和失望。
- 公共场合争吵通常额外提高面子压力和记忆强度。
- 补偿不能替代沟通；只补偿不解决问题会留下旧账。

## 8. 感知反馈生成

玩家不能看到真实值，只能看到模糊反馈。反馈由真实变化、玩家性格、可见信息、过往记忆和当前压力共同生成。

| 反馈级别 | 触发条件 | 示例文本 |
| --- | --- | --- |
| 稳定 | 行为一致、解释清楚、无明显破绽 | “你感觉这次解释基本说得通，气氛没有明显变差。” |
| 轻微不安 | 回复延迟、语气变化、细节不足 | “你说不上哪里不对，但心里还是有一点不舒服。” |
| 明显异常 | 前后说法不一致、回避关键问题、旧账被触发 | “这次解释少了最关键的部分，你感觉对方在绕开什么。” |
| 强烈怀疑 | 破绽明显、共同朋友信息冲突、查证出现线索 | “你还没有完整证据，但几个细节已经对不上了。” |
| 关系危机 | 原则性事件、重大谎言暴露、反复拉黑或公开羞辱 | “这件事已经不只是误会，你们需要决定关系还能不能继续。” |

感知反馈可以与真实值不同。真实只是忙碌时，玩家仍可能轻微不安；真实存在隐瞒时，玩家也可能因为信任高而暂时接受解释。

## 9. 破绽值与暴露判定

破绽值记录隐瞒或谎言可被发现的线索强度。

破绽来源：

- 时间线矛盾。
- 朋友圈照片或动态。
- 共同朋友说法。
- 电话、短信或聊天记录异常。
- 消费记录或地点信息。
- 前后解释不一致。
- 小号、双重账号或两部手机线索。
- 第三者主动出现。

暴露判定规则：

- 破绽低：玩家只能产生轻微不安，无法确认。
- 破绽中：玩家可以通过追问或查证得到部分线索。
- 破绽高：一次追问、共同朋友信息或公开动态可能直接暴露。
- 破绽暴露不等于关系立刻结束，但会显著影响信任、记忆和阶段变化。

## 10. 记忆账本写入规则

不是所有事件都写入记忆账本。写入条件：

- 原则性事件。
- 重大撒谎或隐瞒。
- 已有边界协议仍被破坏。
- 公共场合争吵或羞辱。
- 反复发生的消息延迟、冷战或拉黑。
- 说好私下沟通但长期不解决。
- 有效修复行为，例如完整坦白、道歉、设边界并执行。

记忆写入字段：

- 事件 ID。
- 事件摘要。
- 真实情况。
- 玩家当时感知。
- 相关行为标签。
- 是否解决。
- 情绪残留。
- 后续触发条件。

## 11. 阶段变化判定

阶段变化由多维综合决定，不由单个数值决定。

判断输入：

- 当前阶段。
- 好感、信任、安全感、压力、矛盾、失望。
- 生理吸引和心理喜欢。
- 分手念头。
- 未解决冲突数量和强度。
- 最近 3 天互动趋势。
- 是否发生原则性事件。
- 是否有有效修复行为。

可能变化：

- 升温：互动稳定、主动性提高、真实信任和安全感较稳。
- 确认关系：暧昧阶段下双方投入和边界趋于明确。
- 继续暧昧：好感存在，但信任、规划或边界不足。
- 冷淡：回复变慢、主动性下降、未解决冲突增加。
- 分手危机：重大信任问题、反复拉黑、原则性事件或失望过高。
- 分手：一方明确结束关系，或危机后修复失败。
- 分分合合倾向：强吸引和低信任同时存在，冷战和复联反复出现。

## 12. 原则性事件直达危机规则

以下事件可以绕过慢性累积，直接进入分手危机或显著提高危机权重：

- 重大谎言暴露。
- 明确脚踏两只船暴露。
- 亲密边界严重破坏。
- 公开羞辱。
- 强控制行为。
- 反复拉黑。
- 重要节日/关键危机长期缺席。
- 重大经济欺骗。
- 意外怀孕后严重逃避责任。

规则：

- 只写系统后果，不写露骨内容。
- 成年校验仍是硬规则，亲密和意外怀孕相关机制只适用于成年人。
- 原则性事件进入记忆账本，并影响后续复合难度。
- 有修复机会不代表必然修复，必须看坦白程度、责任承担、边界重建和对方剩余信任。

## 13. v0.1 暂不计算内容

v0.1 暂不计算：

- 完整一年关系曲线。
- 复杂经济资产和负债。
- AI 自然语言解析。
- 完整亲密系统细节。
- 双重账号的完整追踪机制。
- 大规模第三者网络。
- 家庭长期干预线。
- 复杂健康系统。
- UI 展示规则。
- 数据库结构。

v0.1 只需要把 14 天测试版、3 个种子事件、行为标签、模糊反馈、破绽和记忆账本跑通为纸面闭环。

## 14. 心理学行为标签映射

本节把已有行为标签映射到心理学框架。映射只用于解释概率性影响，不代表某个行为必然导致固定结果。

| 行为标签 | 心理学来源 | 关系影响 |
| --- | --- | --- |
| 温和确认 | 非暴力沟通：观察 + 感受 | 降低攻击性，提高修复余量，适合在信息不足时确认事实 |
| 强硬质问 | 焦虑依恋 / 高控制反应 | 可能获得信息，也提高对方压力和防御 |
| 反向指责 | 戈特曼四骑士：辩护 | 延后问题解决，增加未解决冲突和记忆写入概率 |
| 冷处理 | 戈特曼四骑士：筑墙 / 回避依恋 | 降低即时冲突，增加长期失望和安全感下降 |
| 私下沟通 | 修复尝试 / 非暴力沟通 | 提高修复余量，降低公共场合面子压力 |
| 公共场合争吵 | 批评 / 鄙视风险 | 增加面子压力、羞辱感和记忆强度 |

| 分手 | 原则边界 / 失望累积 | 阶段直接变化，后续复合难度取决于记忆和修复 |

使用规则：

- 非暴力沟通标签通常降低攻击性，但不保证对方一定接受。
- 戈特曼四骑士相关标签通常提高长期风险，尤其在重复出现时。
- 依恋反应只表示安全感和距离调节方式，不是角色诊断。
- 修复尝试必须具体，单纯道歉或补偿不能自动清除旧账。
- 玩家看到的是关系反馈，不看到“你触发了某理论”的提示。
