# 关系报告标签词典

本文档用于统一 IF 项目中关系报告、问卷报告、事件复盘和后续 `relationship_state_aggregator` 输出中使用的标签。它把 `docs/design/25_attribution_memory_belief_system.md` 到 `docs/design/35_relationship_event_template_library.md` 中分散出现的标签整理为统一词典。

它不是心理诊断词典，也不是给玩家或 NPC 贴道德标签的工具。它的作用是：

```text
把系统判断翻译成玩家可读、非羞辱、非诊断、可行动的关系反馈。
```

---

## 1. 标签输出原则

### 1.1 标签不是诊断

禁止写法：

```text
你有病态依恋。
你是控制狂。
你是恋爱脑。
你疑心病很重。
你很容易被骗。
你不懂爱。
```

推荐写法：

```text
你对关系中的异常信号较敏锐，这能帮助你更早发现真实风险；但在证据不足时，也可能让你承受更高的关系压力。
```

### 1.2 标签要描述行为模式，不定义人格本质

禁止：

```text
你就是这样的人。
```

推荐：

```text
在压力较高的关系情境中，你可能更容易进入这种回应模式。
```

### 1.3 标签要给出双面性

多数关系倾向都同时有保护作用和风险：

| 倾向 | 保护作用 | 风险 |
| --- | --- | --- |
| 高警觉 | 更早发现问题 | 证据不足时痛苦更高 |
| 高信任 | 关系更稳定 | 可能低估真实风险 |
| 高表露 | 更快建立亲密 | 关系过浅时可能给对方压力 |
| 高隐私边界 | 保留自我空间 | 伴侣可能感到距离 |
| 高共有倾向 | 更愿意照顾对方 | 可能长期忽略自己的公平需求 |

### 1.4 标签要可行动

每个标签最好回答：

```text
这说明什么？
它保护了你什么？
它可能带来什么风险？
你可以怎样调整？
伴侣怎样配合会更好？
```

## 3. 认知与归因标签

### 3.1 `accurate_alertness`：准确警觉

| 字段 | 内容 |
| --- | --- |
| 中文名 | 准确警觉 |
| 来源 | 25、26、31、35 |
| 触发条件 | 玩家/NPC 怀疑有问题，且事实层和证据链支持该怀疑 |
| 正面含义 | 能较早捕捉真实风险，不轻易忽略异常信号 |
| 风险 | 若长期处于高警觉，可能增加关系疲惫 |
| 禁止写法 | “你就是疑心病。” |
| 推荐表达 | “你对异常线索的敏感度较高，并且在这次情境中，这种警觉有一定事实依据。” |

---

### 3.2 `over_suspicion_pattern`：过度怀疑模式

| 字段 | 内容 |
| --- | --- |
| 中文名 | 过度怀疑模式 |
| 来源 | 26、31、34 |
| 触发条件 | 证据链较弱，投射或威胁扫描较强，但角色提前进入负面解释 |
| 正面含义 | 有保护自己免受伤害的功能 |
| 风险 | 容易在证据不足时造成冲突和消耗 |
| 禁止写法 | “你想太多。”、“你不正常。” |
| 推荐表达 | “你在模糊线索下容易优先考虑风险，这能保护你，但也可能让关系在证据不足时提前进入紧张状态。” |

---

### 3.3 `confident_misreader`：自信误读者

| 字段 | 内容 |
| --- | --- |
| 中文名 | 自信误读 |
| 来源 | 26、35 |
| 触发条件 | `partner_model_confidence` 高，但 `partner_model_accuracy` 低 |
| 正面含义 | 关系判断有明确立场，不容易摇摆 |
| 风险 | 可能把投射、旧印象或恐惧当成事实 |
| 禁止写法 | “你很蠢。”、“你根本不懂对方。” |
| 推荐表达 | “你对伴侣的判断较有信心，但部分判断可能来自旧印象、投射或未被验证的推断。” |

---

### 3.4 `selective_blindness_pattern`：选择性忽视

| 字段 | 内容 |
| --- | --- |
| 中文名 | 选择性忽视 |
| 来源 | 26、31 |
| 触发条件 | 事实或证据存在异常，但角色倾向于维持稳定解释，不愿深入确认 |
| 正面含义 | 能降低短期冲突，维持关系稳定感 |
| 风险 | 可能低估真实伤害或长期隐患 |
| 禁止写法 | “你活该被骗。” |
| 推荐表达 | “你倾向于优先维持关系稳定，因此有时会淡化不一致信号。这样能减少短期冲突，但也可能延后面对真实问题。” |

### 3.5 `outdated_partner_model`：旧印象固化

| 字段 | 内容 |
| --- | --- |
| 中文名 | 旧印象固化 |
| 来源 | 26、35 |
| 触发条件 | 角色用过去的伴侣形象解释当前行为，忽略对方变化 |
| 正面含义 | 能从历史模式中识别重复风险 |
| 风险 | 可能不给对方更新和修复机会 |
| 禁止写法 | “你记仇。” |
| 推荐表达 | “你会参考过去的相处经验理解当前事件，这有助于识别重复模式，但也可能让你较难看见对方的新变化。” |

### 3.6 `romantic_idealizer`：浪漫理想化

| 字段 | 内容 |
| --- | --- |
| 中文名 | 浪漫理想化 |
| 来源 | 26 |

## 4. 表露与回应标签

### 4.1 `deep_discloser`：深层表露者

| 字段 | 内容 |
| --- | --- |
| 中文名 | 深层表露者 |
| 来源 | 27、28 |
| 触发条件 | 玩家愿意分享深层经历、脆弱点、家庭/前任/创伤/欲望等内容 |
| 正面含义 | 更容易建立真实亲密 |
| 风险 | 如果关系阶段过浅或对方回应性低，可能更容易受伤 |
| 禁止写法 | “你太情绪化。” |
| 推荐表达 | “你比较愿意把真实经历和脆弱部分带入关系，这能加深亲密，但也需要确认对方是否有能力接住这些内容。” |

---

### 4.2 `guarded_discloser`：谨慎表露者

| --- | --- |
| 中文名 | 谨慎表露者 |

| 风险 | 伴侣可能感到距离、不被信任或难以靠近 |
| 禁止写法 | “你冷漠。” |
| 推荐表达 | “你通常需要足够安全感才会说出深层内容，这能保护你的边界，但也可能让亲密推进速度较慢。” |

### 4.3 `fast_disclosure_pattern`：快速表露模式

| 字段 | 内容 |
| --- | --- |
| 中文名 | 快速表露模式 |
| 来源 | 27、28 |
| 触发条件 | 关系早期即快速透露大量私人或深层信息 |

| 推荐表达 | “你在关系升温时可能较快分享深层内容，这能迅速建立连接，但也需要观察对方是否准备好进入同样深度。” |

### 4.4 `high_responsiveness_need`：高回应需求

| 字段 | 内容 |
| --- | --- |
| 中文名 | 高回应需求 |
| 来源 | 27、28、35 |
| 触发条件 | 说出脆弱或重要内容后，需要明确倾听、安慰、确认或陪伴 |

| 风险 | 遇到低表达或问题解决导向伴侣时，容易感到不被接住 |
| 禁止写法 | “你太需要别人哄。” |
| 推荐表达 | “你在表达脆弱时较需要明确回应和情绪确认。对你来说，被认真听见比单纯给建议更重要。” |

### 4.5 `disclosure_regret`：表露后悔

| 字段 | 内容 |
| --- | --- |
| 中文名 | 表露后悔 |
| 来源 | 27、35 |
| 触发条件 | 角色表露脆弱后遭遇嘲讽、敷衍、攻击或不安全回应 |

| 风险 | 后续更难主动说真话，可能写入旧伤 |
| 禁止写法 | “你玻璃心。” |
| 推荐表达 | “这次表露没有被好好接住，可能让你之后更难再主动说出类似内容。” |

## 5. 秘密与边界标签

### 5.1 `privacy_boundary_conflict`：隐私边界冲突

| 字段 | 内容 |
| --- | --- |
| 中文名 | 隐私边界冲突 |
| 来源 | 27、28、31、35 |
| 触发条件 | 双方对手机、前任、过去经历、金钱、聊天记录等隐私边界期待不同 |
| 正面含义 | 暴露出需要协商的边界差异 |
| 风险 | 若被误解为欺骗或控制，容易升级为信任危机 |
| 禁止写法 | “有隐私就是有问题。” |
| 推荐表达 | “这次问题更像是双方对隐私和透明度的期待不同，需要协商哪些信息属于个人边界，哪些会影响当前关系安全。” |

---

### 5.2 `high_transparency_expectation`：高透明期待

| 字段 | 内容 |
| --- | --- |
| 中文名 | 高透明期待 |
| 来源 | 28 |
| 触发条件 | 角色希望伴侣尽可能坦白手机、前任、异性朋友、金钱等信息 |
| 正面含义 | 有助于减少不确定感 |
| 风险 | 若缺少边界感，可能让伴侣感到被审查或控制 |
| 禁止写法 | “你控制欲强。” |
| 推荐表达 | “你对关系透明度的期待较高，这能降低不确定感，但需要和伴侣区分‘必要坦白’和‘正当隐私’。” |

### 5.3 `deception_risk`：欺骗风险

| 字段 | 内容 |
| --- | --- |
| 中文名 | 欺骗风险 |
| 来源 | 25、27、35 |
| 触发条件 | 存在前后解释不一致、隐藏证据、删除痕迹、半真半假、主动撒谎等线索 |
| 正面含义 | 帮助系统标记需要进一步证据校验的高风险事件 |
| 风险 | 不能在证据不足时滥用 |
| 禁止写法 | “对方一定是渣。” |
| 推荐表达 | “当前事件存在解释与线索不一致的部分，关系系统会将其标记为需要后续校验的信任风险。” |

## 6. 冲突与修复标签

### 6.1 `active_listener`：积极倾听者

| 字段 | 内容 |
| --- | --- |
| 中文名 | 积极倾听者 |
| 来源 | 29 |
| 触发条件 | 能复述、澄清、确认对方感受，不急于打断或反驳 |
| 正面含义 | 有利于冲突降温和关系修复 |
| 风险 | 若只听不表达自身需求，也可能形成压抑 |
| 禁止写法 | 无 |
| 推荐表达 | “你在冲突中有能力先确认对方真正想表达的内容，这会明显提高修复机会。” |

---

### 6.2 `repair_capable`：具备修复能力

| 字段 | 内容 |
| --- | --- |
| 中文名 | 修复能力较强 |
| 来源 | 29、34、35 |
| 触发条件 | 冲突后能道歉、补偿、确认感受、提出具体调整或暂停后回来继续谈 |
| 正面含义 | 有助于关系从冲突中恢复 |
| 风险 | 若总是单方修复，可能形成不公平 |
| 禁止写法 | “会道歉就没问题。” |
| 推荐表达 | “这段关系中存在一定修复能力。即使发生冲突，双方仍有机会通过具体回应和后续行动恢复稳定。” |

---

### 6.3 `stonewalling_pattern`：石墙模式

| 字段 | 内容 |
| --- | --- |
| 中文名 | 石墙模式 |
| 来源 | 29、31、34、35 |
| 触发条件 | 长期拒绝回应、冷处理、用沉默惩罚或控制，而不是有效暂停 |
| 正面含义 | 有时源于情绪过载，需要冷静空间 |
| 风险 | 若不回到问题，会严重降低修复机会 |
| 禁止写法 | “沉默就是冷暴力。” |
| 推荐表达 | “当前更接近回避性沉默或石墙：问题没有被回应，也没有明确的修复窗口，因此会削弱安全感和修复机会。” |

## 11. 标签组合规则

### 11.1 不要输出互相矛盾标签

禁止同一事件同时输出：

```text
accurate_alertness + over_suspicion_pattern
trust_damage_event + short_term_fluctuation
stonewalling_pattern + timeout_repairer
safe_but_bored_pattern + risk_excitement_pattern
```

除非明确分不同阶段或不同角色。

### 11.2 主标签和副标签

建议每次事件最多输出：

```text
1 个主标签
1-3 个副标签
```

避免报告变成标签堆砌。

### 11.3 标签强度

建议标签保留强度：

```text
low / medium / high
```

示例：

```json
{
  "tag": "privacy_boundary_conflict",
  "strength": "medium",
  "source_event": "E-SEC-01"
}
```

## 12. 报告输出模板

### 12.1 单事件复盘模板

```text
这次事件主要影响的是【信任 / 亲密 / 满意度 / 公平感 / 修复机会】。

从事实层看：……
从解释层看：……
从沟通层看：……
从长期影响看：……

当前更接近的模式是：{主标签中文名}。
这不代表你/对方“就是这样的人”，而是说明在这个情境下，关系系统观察到了这种互动倾向。
```

### 12.2 问卷报告模板

```text
你的关系倾向中，较明显的一点是：{标签中文名}。

这通常意味着：……
它能保护你：……
也可能带来：……
更适合你的相处方式是：……
需要留意的关系场景是：……
```
