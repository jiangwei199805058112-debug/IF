- `docs/design/15_full_questionnaire_upgrade_plan.md`
- `docs/design/16_questionnaire_dimension_table.md`
- `docs/design/17_question_type_schema.md`
- `docs/design/18_super_realistic_question_bank.md`

本文档只做报告结构设计，不修改控制台原型运行逻辑。

## 1. 报告设计目标

问卷报告不是心理诊断，也不是现实人格定罪。报告的目标是把玩家的自述答案整理成：

1. 可读的关系画像；
2. 可解释的关系风险；
3. 可进入游戏系统的初始档案；
4. 可被后续游戏行为修正的自述人格；
5. 可用于 NPC 反应、事件权重、信任变化、谎言破绽和结局标签的输入。

报告应避免：

- 你就是某种病态人格；
- 你一定会出轨；
- 你就是渣男/渣女；
- 你没有改变可能；
- 单凭一题给出严重结论。

报告应使用行为层表述：

```text
你在压力下更倾向于选择性表达，而不是立刻完整坦白。
你对自己的私人空间需求较高，但对对方的异性社交透明度也有较高期待。
这意味着你在边界协商中可能需要明确“双方同一规则”。
```

## 2. 报告总结构

超级真实版报告建议由以下部分组成。

| 顺序 | 模块 | 用途 |
| --- | --- | --- |
| 1 | 封面摘要 | 给玩家快速看懂整体画像 |
| 2 | 问卷完整度与可信度 | 告知报告可信范围 |
| 3 | 主画像 | 一句话概括核心关系模式 |
| 4 | 关键维度雷达 | 展示核心维度高低倾向 |
| 5 | 亲密与依恋报告 | 解释亲密、独立、安全感 |
| 6 | 信任与边界报告 | 解释怀疑、查证、占有和隐私 |
| 7 | 诚实与信息管理报告 | 解释坦白、隐瞒、半真半假 |
| 8 | 冲突与修复报告 | 解释冷战、爆发、道歉、复盘 |
| 9 | 欲望与忠诚报告 | 解释新鲜感、诱惑、替代倾向 |
| 10 | 现实压力报告 | 解释时间、金钱、家庭、作息压力 |
| 11 | 数字生活报告 | 解释在线、手机、社交媒体和证据链 |
| 12 | 自述人格 vs 行为风险 | 检测自我美化和潜在矛盾 |
| 13 | 高风险关系场景 | 生成最容易出问题的事件列表 |
| 14 | 适合/不适合的相处模式 | 给出关系运行建议 |
| 15 | 游戏内初始标签 | 转成系统可用标签 |
| 16 | 后续行为修正提示 | 告诉玩家游戏中会继续更新画像 |

## 3. 报告数据输入

报告输入建议使用以下结构。

```text
questionnaire_report_input:
  mode: super_realistic
  total_questions: 150
  answered_questions: 150
  skipped_questions: 0
  completion_rate: 1.0
  answer_consistency_score: 0.78
  self_beautification_risk: medium
  double_standard_risk: medium_high
  dimension_scores:
    attachment_abandonment_anxiety: 76
    attachment_intimacy_avoidance: 54
    trust_checking_impulse: 68
    info_concealment_tendency: 61
    communication_repair_initiative: 42
  strong_tags:
    - high_reassurance_need
    - pressure_based_concealment
  weak_evidence_dimensions:
    - family_privacy_boundary
    - stability_self_care
```

## 4. 完整度与可信度模板

### 4.1 完整度等级

| 完成率 | 等级 | 文本 |
| ---: | --- | --- |
| 90%-100% | 高 | 你完成了大部分题目，报告可作为较完整的自述画像。 |
| 70%-89% | 中高 | 报告可以反映主要倾向，但部分模块证据不足。 |
| 50%-69% | 中 | 报告只能作为粗略画像，后续游戏行为会更明显地修正。 |
| 25%-49% | 低 | 报告可信度有限，适合快速开始而非真实模拟。 |
| 0%-24% | 很低 | 无法形成稳定画像，只能使用默认角色倾向。 |

### 4.2 一致性等级

| 一致性 | 文本 |
| --- | --- |
| 高 | 你的自评题、情境题和反向题大体一致。 |
| 中 | 你的答案存在一些情境差异，说明你在不同压力下可能表现不同。 |
| 低 | 你的自评和情境选择差异明显，报告会降低部分结论的确定度。 |

### 4.3 可信度输出示例

```text
你完成了150题中的147题，问卷完整度为98%。
你的答案整体一致性为中高。部分题目显示：你自认为尊重隐私，但在不安场景中仍有较强查证冲动。
因此，系统会把“边界尊重”视为自述高值，但把“压力下查证风险”作为游戏内潜在风险保留。
```

## 5. 主画像模板

主画像用于一段话概括玩家的关系模式。

### 5.1 主画像结构

```text
你在亲密关系中的核心模式是：
【亲密需求】 + 【安全感来源】 + 【压力下反应】 + 【主要风险】 + 【修复方式】。
```

### 5.2 主画像示例

#### 类型 A：高安全感修复型

```text
你更接近“稳定沟通型”。你通常愿意信任对方，也能在关系出现问题时通过解释、复盘和补偿来修复。你不太依赖查证或冷处理获得安全感。你的主要风险不是戏剧性冲突，而是可能低估对方的不安，需要在重要节点主动表达。
```

#### 类型 B：焦虑确认型

```text
你更接近“高确认需求型”。你很在意对方是否稳定回应，也容易从延迟、模糊解释和异性社交中感到不安。你不一定想控制对方，但在压力下可能通过追问、查证或反复确认来获得安全感。你的关系重点是把“确认需求”表达成具体请求，而不是变成暗中查证或情绪试探。
```

#### 类型 C：回避保护型

```text
你更接近“距离保护型”。你需要亲密，但也很重视个人空间。关系推进太快、对方过度依赖或频繁追问时，你可能会本能后退。你的风险是把退缩包装成冷静，让对方感到被冷落。你的关系重点是提前说明自己的空间需求，而不是临时消失。
```

#### 类型 D：拉扯矛盾型

```text
你更接近“亲密拉扯型”。你既想被稳定选择，也害怕失去自由或暴露真实脆弱。你可能在一段关系里同时表现出靠近和退缩：不安时想确认，压力大时又想逃开。你的关系重点是识别自己什么时候是在表达需要，什么时候是在用冷淡或试探保护自己。
```

#### 类型 E：高隐瞒压力型

```text
你更接近“压力选择性表达型”。你并不一定想欺骗对方，但当你预判某件事会引发冲突时，容易选择说一部分、晚点说或模糊带过。你的关系风险在于：你以为自己是在避免争吵，对方却可能把它理解成不诚实。你的关系重点是区分“隐私”和“关键事实”。
```

#### 类型 F：新鲜感高需求型

```text
你更接近“高刺激需求型”。稳定关系能给你安全感，但重复和缺少情绪回应时，你容易被新鲜感、被理解感或外部欣赏吸引。你的风险不是必然背叛，而是在关系不顺时给第三方留下过多入口。你的关系重点是主动经营新鲜感，而不是等关系变淡后向外寻找补足。
```

## 6. 关键维度摘要模板

报告不建议展示128个维度全部分数。建议展示 12-20 个关键维度。

### 6.1 推荐展示维度

| 模块 | 维度 |
| --- | --- |
| 依恋 | 忧虑被弃、回避亲密、亲密需求、独立需求 |
| 信任 | 基础信任、查证冲动、解释接受度、旧账记忆 |
| 边界 | 占有欲、边界尊重、双标倾向、控制欲 |
| 信息 | 诚实倾向、隐瞒倾向、半真半假倾向 |
| 冲突 | 直接沟通、冷处理、防御性、修复主动性 |
| 欲望 | 新鲜感需求、诱惑抵抗、替代倾向 |
| 现实 | 时间管理、金钱压力、家庭影响、作息稳定 |

### 6.2 展示格式

```text
关键倾向：
- 安全感需求：较高
- 查证冲动：中高
- 隐瞒倾向：中
- 冲突修复主动性：中低
- 新鲜感需求：中高
- 双标风险：中
```

不要显示：

```text
attachment_abandonment_anxiety = 76
```

除非处于开发者调试模式。

## 7. 模块报告模板

### 7.1 亲密与依恋报告

```text
你对亲密关系的需求较强，但也需要一定个人空间。你希望对方稳定回应，在关系推进变快时又可能出现压力感。这种组合意味着：你并不是单纯黏人或单纯冷淡，而是在“想靠近”和“怕失控”之间切换。

游戏内影响：
- 消息延迟更容易触发不安；
- 对方忽冷忽热会放大你的旧伤记忆；
- 关系推进太快时，你可能需要降速选项；
- 真诚解释和稳定行动能降低你的不安。
```

### 7.2 信任与边界报告

```text
你的信任不是完全低，而是对“解释不完整”和“边界模糊”非常敏感。你可以接受对方有私人空间，但当异性社交、前任联系或手机隐私叠加出现时，你会更想确认细节。

游戏内影响：
- 神秘电话、手机遮屏、朋友圈隐藏会提高怀疑；
- 对方主动说明会明显降低误会；
- 如果你选择暗中查证，隐私信任会下降；
- 如果你要求对方透明但自己保留空间，双标标签会上升。
```

### 7.3 诚实与信息管理报告

```text
你倾向于把诚实和冲突成本放在一起考虑。你不一定会直接撒谎，但在预判对方会生气时，可能选择晚点说、说一半或避开细节。

游戏内影响：
- 小事隐瞒可能先降低冲突，但增加后续破绽；
- 半真半假会产生证据链风险；
- 被揭穿时如果继续解释不完整，事实信任会下降；
- 主动坦白会短期引发情绪，但长期更利于修复。
```

### 7.4 冲突与修复报告

```text
你在冲突中既有表达需求，也有防御或回避倾向。被质疑时，你可能先解释自己为什么这样做，而不是先承接对方感受。冲突后是否能修复，取决于你能不能从“证明我没错”转向“处理对方受伤”。

游戏内影响：
- 直接解释可能被对方理解为辩解；
- 冷处理会增加未解决冲突；
- 真诚道歉、具体补救和复盘会打开修复路线；
- 反复说重话会形成长期关系伤口。
```

### 7.5 欲望与忠诚报告

```text
你对关系中的新鲜感和情绪价值比较敏感。当现有关系缺少理解、赞美或陪伴时，外部对象的关注更容易对你产生吸引力。你的忠诚风险不一定来自主动寻找背叛，而可能来自“关系不顺时没有及时关门”。

游戏内影响：
- 第三方安慰会提高精神暧昧风险；
- 主动设边界可以降低诱惑事件；
- 隐藏聊天会增加信息破绽；
- 稳定关系中主动制造新鲜感能降低外部吸引。
```

### 7.6 现实压力报告

```text
现实压力会显著影响你的关系表现。时间、作息、金钱或家庭压力上升时，你可能减少联系、降低解释耐心，甚至把关系问题暂时推后处理。

游戏内影响：
- 高压期消息延迟不一定代表感情下降；
- 但如果没有提前说明，对方感知值会下降；
- 经济压力会影响约会、礼物和仪式感事件；
- 家庭压力可能改变承诺节奏和公开程度。
```

### 7.7 数字生活报告

```text
你对数字生活中的细节比较敏感。在线未回、朋友圈分组、点赞评论、聊天记录和小号都可能成为你判断关系状态的线索。数字痕迹会让你更容易确认，也更容易误判。

游戏内影响：
- 社交媒体互动会影响忠诚信任；
- 在线不回会影响情感信任；
- 删除聊天记录会增加证据链风险；
- 双账号和分组可见会触发身份/透明度事件。
```

## 8. 高风险关系场景模板

高风险场景不应超过 5 个，避免报告吓人。

### 8.1 输出格式

```text
你最容易出问题的场景：
1. 对方解释不完整 + 手机行为异常
   - 你可能先忍住，但内心会开始查证。
   - 建议：优先选择“温和确认事实”，不要直接进入暗查。
2. 关系不顺 + 出现能理解你的人
   - 你可能把情绪价值转移到第三方。
   - 建议：先处理现有关系边界，再决定是否退出。
3. 工作/学习高压 + 对方要求陪伴
   - 你可能觉得被施压，对方则觉得被冷落。
   - 建议：提前说明高压期联系规则。
```

### 8.2 高风险场景候选池

| 场景 | 触发条件 |
| --- | --- |
| 消息延迟误会 | 高忧虑被弃 + 高在线可得压力 |
| 异性饭局危机 | 高忠诚信任敏感 + 高隐瞒倾向 |
| 神秘电话危机 | 高怀疑敏感 + 高解释接受度低 |
| 手机隐私冲突 | 高查证冲动 + 高手机隐私需求 |
| 冷战失联 | 高情绪关机 + 高冲突回避 |
| 说重话伤口 | 高冲突伤人 + 高旧账记忆 |
| 精神暧昧 | 高情绪价值饥渴 + 低诱惑抵抗 |
| 关系替代 | 高替代倾向 + 高新鲜感需求 |
| 家庭压力 | 高家庭影响 + 高未来规划需求 |
| 现实冷淡误判 | 高现实压力 + 高亲密需求 |

## 9. 适合/不适合相处模式模板

### 9.1 适合模式

```text
更适合你的关系模式：
- 有稳定回应，但不过度控制；
- 重要社交边界提前说明；
- 冲突后可以复盘，而不是只翻篇；
- 允许双方保留隐私，但关键事实不隐瞒；
- 在高压期提前设定联系规则；
- 稳定中仍保留约会、新鲜感和情绪表达。
```

### 9.2 不适合模式

```text
不太适合你的关系模式：
- 长期忽冷忽热；
- 解释模糊但要求你无条件信任；
- 对方边界不清，又拒绝讨论规则；
- 你需要确认时，对方只说“你想太多”；
- 冲突后长期冷处理；
- 关系稳定后完全失去情绪交流和新鲜感。
```

## 10. 游戏内初始标签模板

报告要转成游戏系统可用标签。标签分为可见标签和隐藏标签。

### 10.1 玩家可见标签

```text
可见标签：
- 需要稳定回应
- 重视边界说明
- 压力下容易回避
- 修复需要具体行动
- 重视精神共鸣
```

### 10.2 系统隐藏标签

```text
hidden_tags:
  - high_reassurance_need
  - suspicion_under_ambiguity
  - pressure_based_concealment
  - conflict_shutdown_risk
  - emotional_validation_vulnerability
```

### 10.3 标签使用原则

- 可见标签用于玩家报告；
- 隐藏标签用于事件权重；
- 高风险标签需要多题支持；
- 游戏行为可以覆盖或修正问卷标签；
- 同一标签可随剧情变化增强或减弱。

## 11. 自述人格 vs 行为人格模板

问卷报告只能生成 `self_report_profile`。游戏过程中需要生成 `behavior_profile`。

### 11.1 报告文本

```text
当前报告基于你的自述答案生成。游戏开始后，系统会继续记录你的实际选择。

如果你在问卷中表示“尊重隐私”，但游戏中多次选择查证、偷看或逼问，系统会逐渐提高“压力下查证风险”。
如果你在问卷中表示“重视诚实”，但游戏中多次选择隐瞒或半真半假，系统会提高“选择性表达倾向”。
```

### 11.2 数据结构

```text
profile_gap:
  self_report_profile:
    boundary_respect: high
    honesty_tendency: high
  behavior_profile:
    checking_impulse: medium_high
    partial_truth_tendency: high
  gap_tags:
    - self_beautification_risk
    - stress_behavior_shift
```

## 12. 报告模板样例

### 12.1 完整样例：高确认需求 + 选择性表达

```text
# IF 关系问卷报告

## 你的主画像

你更接近“高确认需求 + 压力选择性表达型”。
你很重视稳定回应和清楚的关系边界。对方延迟回复、解释不完整或异性社交模糊时，你容易产生不安，并希望通过追问、确认或观察线索来恢复安全感。

同时，你在自己面对可能引发冲突的信息时，并不一定会第一时间完整坦白。你更可能先判断“说出来会不会吵架”，再决定说多少。这会形成一种关系矛盾：你希望对方透明，但自己在压力下也会保留空间。

## 报告可信度

你完成了150题中的150题，问卷完整度高。你的答案一致性为中高。系统发现你在“隐私边界”和“查证冲动”之间存在一定张力，因此相关结论会在游戏中继续观察。

## 关键倾向

- 安全感需求：较高
- 查证冲动：中高
- 解释接受度：中
- 隐瞒倾向：中高
- 双标风险：中
- 修复主动性：中
- 新鲜感需求：中
- 现实压力影响：中高

## 你最容易出问题的场景

1. 对方解释不完整
   你可能表面冷静，但心里开始寻找更多线索。

2. 自己做了容易被误会的事
   你可能先说一部分，等对方追问再补充。

3. 高压期联系减少
   你会希望对方理解，但如果没有提前说明，对方可能感到被冷落。

## 更适合你的相处方式

你更适合“稳定回应 + 清楚边界 + 允许隐私但不隐瞒关键事实”的关系。你需要的不是对方完全透明，而是重要社交和容易误会的事情提前说明。

## 游戏内影响

- 模糊解释事件更容易触发不安；
- 你可以通过温和确认降低误会；
- 如果选择暗中查证，会损害隐私信任；
- 如果选择主动坦白，短期可能有冲突，但长期更利于事实信任。
```

### 12.2 简短样例：稳定修复型

```text
你更接近“稳定修复型”。你通常愿意先相信对方，也有较强的复盘和修复能力。你不太依赖查证获得安全感，但可能低估对方对表达和仪式感的需要。

游戏内影响：
- 大多数误会可以通过解释修复；
- 冷处理风险较低；
- 如果长期忽视表达，对方可能感到你不够在意；
- 适合稳定推进、边界清楚、低戏剧性的关系线。
```

### 12.3 简短样例：高自由需求型

```text
你更接近“高自由需求型”。你需要亲密，但不喜欢被过度追问、报备或控制。你对私人空间和自主性要求较高。你的风险是：当对方需要确认时，你可能把它理解成控制，从而后退或冷处理。

游戏内影响：
- 报备和查手机事件容易引发反感；
- 关系推进过快会增加压力；
```
