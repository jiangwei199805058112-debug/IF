# 社会交换、比较水平、替代选择与依赖权力系统设计

本文档用于沉淀 IF 项目中的社会交换、关系收益/代价、比较水平、替代选择、依赖度和关系权力机制。它不是普通关系心理学资料整理，而是用于支撑关系满意度、稳定性、离开倾向、替代对象诱惑、现实约束、投入损失和权力不平衡的系统设计。

核心原则：

```text
亲密关系中的满意与稳定不是同一件事。
一个人可能幸福且稳定，也可能不幸福但稳定，还可能幸福但不稳定，甚至既不幸福也不稳定。
关键取决于：当前关系结果、个人期待水平 CL、替代选择水平 CLalt、投入损失和依赖差异。
```

本设计参考亲密关系中的社会交换、相互依赖、奖赏与代价、比较水平、替代比较水平、关系投入、依赖度和较小利益原则等观点，并转译为 IF 的系统机制。本文档不收录原始截图或长篇原文，只保留可供项目使用的变量、事件模板、报告标签和落地边界。

---

## 1. 设计目的

IF 当前已经沉淀了：

- 真实原因层与解释可信度；
- 伴侣认知准确度与危险知觉；
- 沟通、自我表露与秘密边界；
- 冲突沟通与修复回应。

这些系统能解释“为什么会误解”“为什么会受伤”“如何沟通和修复”。但还缺少一个现实关系中非常关键的问题：

```text
一个人为什么继续留在一段关系里，或者为什么离开？
```

现实中，离开并不只取决于爱不爱、满不满意。还取决于：

- 当前关系能给多少收益；
- 当前关系带来多少代价；
- 这个人觉得自己理应得到什么；
- 外面有没有更好的替代对象或生活方式；
- 已经投入了多少时间、金钱、共同圈子、承诺和身份；
- 离开的现实代价有多高；
- 双方谁更依赖谁。

因此 IF 不应简单写成：

```text
满意度高 → 留下
满意度低 → 离开
```

更合理的是：

```text
关系结果 = 奖赏 - 代价
满意度 = 关系结果 - 比较水平 CL
依赖度 = 关系结果 - 替代比较水平 CLalt + 投入损失 + 离开成本
权力差异 = 双方依赖度差异
```

## 2. 系统一：奖赏、代价与关系结果

### 2.1 设计原则

亲密关系中的奖赏不只是物质好处，也包括情绪、陪伴、理解、性吸引、社会支持、身份认同、被照顾感和共同未来。代价也不只是花钱或时间，还包括心理负担、焦虑、失去自由、忍受缺点、现实压力、被拖累、社交损耗和机会成本。

### 2.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `relationship_rewards` | 当前关系总体奖赏 |
| `relationship_costs` | 当前关系总体代价 |
| `relationship_outcome` | 当前关系结果，奖赏减去代价 |
| `emotional_reward` | 情绪价值、安慰、陪伴、被理解 |
| `practical_reward` | 现实帮助、资源支持、生活协作 |
| `romantic_reward` | 浪漫、吸引、亲密体验 |
| `social_reward` | 共同圈子、身份认可、家庭认可 |
| `growth_reward` | 让自己变好、被鼓励、共同成长 |
| `emotional_cost` | 焦虑、委屈、被消耗、担心不确定 |
| `practical_cost` | 金钱、时间、精力、通勤、生活负担 |
| `freedom_cost` | 失去自由、被控制、被限制 |
| `opportunity_cost` | 因维持关系而错过其他可能性 |
| `regret_cost` | 因关系而放弃某些东西后的遗憾 |

### 2.3 事件模板

事件：玩家为了陪伴对方，放弃了一个重要聚会。

可能影响：

- 如果对方感激、回应、之后补偿：`emotional_reward` 上升；
- 如果对方认为理所当然：`regret_cost` 和 `taken_for_granted_signal` 上升；
- 如果玩家本来就重视个人空间：`freedom_cost` 上升；
- 如果这是长期模式：`relationship_costs` 累积，并影响满意度。

## 3. 系统二：比较水平 CL 与满意度

### 3.1 设计原则

比较水平 CL 是一个人认为自己在人际关系中“理应得到”的结果水平。它受过去经历、家庭脚本、自尊、恋爱经验、同伴比较、文化环境和个人期待影响。

同样的关系结果，对不同人可能产生完全不同的满意度。

```text
满意度 = 关系结果 - CL
```

如果关系结果超过 CL，人会感到满足；如果关系结果低于 CL，即使这段关系在外人看来不错，当事人仍可能不满意。

### 3.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `comparison_level` | CL，个人期待收益水平 |
| `expected_relationship_standard` | 认为亲密关系应该达到的标准 |
| `past_relationship_baseline` | 过去关系形成的参考线 |
| `family_model_baseline` | 原生家庭或成长经历形成的关系期待 |
| `self_worth_expectation` | 自尊和自我价值影响下的关系期待 |
| `social_comparison_pressure` | 与朋友、社交媒体或同龄人比较形成的压力 |
| `relationship_satisfaction` | 当前满意度 |
| `gratitude_buffer` | 对当前关系优点的感激缓冲 |
| `entitlement_pressure` | “我应该得到更多”的压力 |

### 3.3 游戏表现

同一 NPC 行为，在不同 CL 玩家身上效果不同。

例子：对方每天晚上回复并偶尔见面。

| 玩家 CL | 解释 |
| --- | --- |
| 低 CL | “已经很好了，对方很稳定。” |
| 中 CL | “还可以，但希望多一点主动。” |
| 高 CL | “这不够，我需要更高投入。” |

### 3.4 报告提示

报告不应写：

```text
你的期待太高。
```

更好的写法：

```text
你对亲密关系的期待水平较高，因此即使关系中有稳定和实际投入，你仍可能更敏感地感到“不够”。这不一定是错误，但它会提高伴侣需要满足的关系标准。
```

## 4. 系统三：替代比较水平 CLalt 与稳定性

### 4.1 设计原则

替代比较水平 CLalt 指一个人认为自己离开当前关系后，能够从其他伴侣、单身生活、朋友支持、事业发展或自由状态中获得的结果。

CLalt 决定的不是满意度，而是依赖度和离开倾向。

```text
依赖度 = 当前关系结果 - CLalt + 投入损失 + 离开成本
```

如果一个人对当前关系不满意，但认为没有更好的替代选择，关系仍可能稳定。如果一个人对当前关系满意，但认为外面有明显更好的选择，关系也可能不稳定。

### 4.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `comparison_level_alternatives` | CLalt，替代选择水平 |
| `alternative_partner_attraction` | 替代伴侣吸引力 |
| `single_life_attraction` | 单身生活吸引力 |
| `career_or_freedom_alternative` | 事业、自由、迁移等非伴侣替代收益 |
| `perceived_dating_market_value` | 自认为在择偶市场中的吸引力 |
| `alternative_information_access` | 获取替代选择信息的机会 |
| `replacement_confidence` | 相信自己能找到更好关系的信心 |
| `loneliness_cost` | 离开后孤独成本 |
| `alternative_uncertainty` | 替代选择不确定性 |
| `leave_tendency` | 离开倾向 |
| `relationship_dependence` | 对当前关系的依赖度 |

### 4.3 游戏表现

| 当前关系 | CLalt | 可能状态 |
| --- | --- | --- |
| 满意 | 低 | 幸福且稳定 |
| 满意 | 高 | 幸福但不稳定 |
| 不满意 | 低 | 不幸福但稳定 |
| 不满意 | 高 | 不幸福且不稳定 |

### 4.4 事件模板

事件：玩家在社交场合遇到一个很有吸引力的新对象。

影响取决于：

- 当前关系结果；
- 当前关系满意度；
- 玩家 CLalt；
- 玩家替代对象吸引力；
- 玩家对当前关系的投入损失；
- 玩家道德和承诺边界；
- 当前关系是否处于冲突期。

如果当前关系结果低、CLalt 高、投入损失低，则诱惑影响很强。如果当前关系结果高、投入深、承诺稳定，则诱惑可能只形成短暂波动。

## 5. 系统四：投入损失与离开成本

### 5.1 设计原则

人不会只根据当前快乐程度决定是否离开关系。已经投入的东西会形成离开成本，包括有形和无形两类。

有形投入：

- 时间；
- 金钱；
- 共同住所；
- 共同物品；
- 共同宠物；
- 共同计划；
- 家庭和朋友圈。

无形投入：

- 共同记忆；
- 习惯；
- 身份认同；
- 被家人认可；
- 对未来的承诺；
- 对对方家人的感情；
- 分开后的羞耻、解释压力和社交损失。

### 5.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `relationship_investment` | 当前关系投入总量 |
| `sunk_cost_pressure` | 沉没成本压力 |
| `shared_history_weight` | 共同记忆重量 |
| `shared_social_network` | 共同社交圈绑定 |
| `family_entanglement` | 双方家庭牵连 |
| `cohabitation_cost` | 同居、搬家、财务拆分成本 |
| `future_plan_investment` | 共同未来规划投入 |
| `identity_investment` | 把自己身份与关系绑定的程度 |
| `breakup_cost` | 分手现实成本 |
| `staying_due_to_cost` | 因离开成本而维持关系 |

### 5.3 事件模板

事件：玩家明明不满意，但想到已经在一起很久、朋友都认识、房租也在一起承担。

系统判断：

```text
不满意度是否高？
CLalt 是否足够高？
投入损失是否压制了离开倾向？
留下是因为修复希望，还是因为离开成本？
```

## 6. 系统五：依赖差异与关系权力

### 6.1 设计原则

在关系中，谁更依赖这段关系，谁往往更缺少权力。较少依赖的一方更容易主导关系，因为对他/她来说，离开损失更小。

这并不一定等于恶意控制，但会形成现实权力差异。

### 6.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `player_dependence` | 玩家对关系的依赖度 |
| `npc_dependence` | NPC 对关系的依赖度 |
| `dependence_gap` | 双方依赖差异 |
| `power_balance` | 关系权力平衡 |
| `lesser_interest_advantage` | 较小利益一方的权力优势 |
| `fear_of_loss` | 害怕失去关系的程度 |
| `neediness_signal` | 需求过高、害怕被抛弃的信号 |
| `take_for_granted_risk` | 对方因你不会离开而理所当然的风险 |
| `concession_pressure` | 为维持关系而让步的压力 |
| `exit_threat_power` | 用离开威胁影响对方的能力 |

### 6.3 游戏表现

| 情况 | 关系权力倾向 |
| --- | --- |
| 玩家很依赖，NPC 不太依赖 | NPC 更有权力，玩家更容易让步 |
| NPC 很依赖，玩家不太依赖 | 玩家更有权力，NPC 更容易焦虑或讨好 |
| 双方都高依赖 | 稳定但可能纠缠、难分开 |
| 双方都低依赖 | 关系脆弱，轻微冲突也可能散掉 |

### 6.4 事件模板

事件：一方说“你不接受就分手”。

系统判断：

- 说这句话的人是否真的有离开能力；
- 对方的依赖度是否更高；
- 这是边界表达，还是权力威胁；
- 过去是否反复使用分手威胁；
- 被威胁方是否因害怕失去而让步。

## 7. 关系四种大状态

综合当前关系结果、CL 和 CLalt，可以得到四种大状态：

| 状态 | 条件 | 游戏含义 |
| --- | --- | --- |
| 幸福且稳定 | 关系结果高于 CL，也高于 CLalt | 满意且依赖当前关系，离开倾向低 |
| 不幸福但稳定 | 关系结果低于 CL，但高于 CLalt | 不满意，但替代选择更差或离开成本高 |
| 幸福但不稳定 | 关系结果高于 CL，但低于 CLalt | 目前满意，但外部选择更有吸引力 |
| 不幸福且不稳定 | 关系结果低于 CL，也低于 CLalt | 不满意且认为外面更好，离开倾向高 |

注意：

```text
幸福/不幸福看 CL；
稳定/不稳定看 CLalt、投入损失和离开成本。
```

## 10. 报告标签设计

| 标签 ID | 含义 | 报告表达方向 |
| --- | --- | --- |
| `high_comparison_level` | 高期待水平 | 对关系质量要求较高，不容易因为“还不错”就满足 |
| `low_comparison_level` | 低期待水平 | 容易满足，但也可能低估自己应得的关系质量 |
| `high_clalt_awareness` | 高替代选择意识 | 会关注外部选择，关系满意不代表绝对稳定 |
| `low_clalt_dependence` | 低替代选择依赖 | 即使不满意，也可能因替代选择不足而维持关系 |
| `investment_bound` | 投入绑定 | 共同历史、圈子、现实成本会显著影响离开决策 |
| `sunk_cost_stayer` | 沉没成本留守 | 可能因为已经投入太多而继续维持不满意关系 |
| `alternative_sensitive` | 替代诱惑敏感 | 外部吸引力出现时，关系稳定性更容易波动 |
| `dependence_imbalance_risk` | 依赖不平衡风险 | 双方谁更怕失去，谁更容易让步或被动 |
| `lesser_interest_power` | 较小利益权力 | 较不依赖的一方可能拥有更多关系话语权 |
| `fear_of_loss_pattern` | 失去恐惧 | 害怕分手可能导致过度让步或压抑需求 |

## 11. 事件库方向

后续可以设计以下事件：

1. 玩家觉得“对方其实不错，但我总觉得不够”；
2. NPC 提供稳定陪伴，但浪漫奖赏不足；
3. 玩家遇到一个更有吸引力的新对象；
4. 双方长期不幸福，但因为同居和共同圈子难以分开；
5. 一方用“分手”作为威胁；
6. 一方为了关系不断让步；
7. 玩家发现自己只是害怕孤独，不是真的想留下；
8. NPC 觉得自己在这段关系里付出更多；
9. 玩家因为过去糟糕恋爱而降低期待；
10. NPC 因自尊低而低估自己的替代选择。

每个事件都应区分：

```text
当前关系结果是多少？
玩家 CL 是多少？
玩家 CLalt 是多少？
投入损失和离开成本是多少？
双方依赖是否不平衡？
留下是因为满意、依赖、习惯、现实成本，还是没有替代？
```

## 12. 不做什么

本系统暂不做：

1. 不把关系写成冷冰冰的经济交易；
2. 不把满意度等同于稳定性；
3. 不把离开倾向直接等同于不爱；
4. 不把留下直接等同于爱；
5. 不把低替代选择写成必然弱势，需要结合投入、承诺和现实资源；
6. 不用单一事件判断关系是否该结束；
7. 不做心理诊断；
8. 不上传原始截图或大段原文；
9. 不接 AI API。

## 13. 一句话总结

```text
IF 的社会交换系统要模拟的不是“谁赚谁亏”，而是：
玩家和 NPC 如何在奖赏、代价、期待、替代选择、投入损失和依赖差异之间判断这段关系是否值得继续。
```
