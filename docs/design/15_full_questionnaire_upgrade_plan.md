# 完整调查问卷升级方案 v0.1

本文档用于记录 IF 开局调查问卷的下一阶段升级方案。目标是把现有短问卷扩展为可长期支撑真实关系模拟的完整建档系统。

当前文档只做设计沉淀，不要求立即实现代码。后续 Codex 或人工开发时，应优先参考本文档、`docs/design/16_questionnaire_dimension_table.md` 和 GitHub issue #1。

## 1. 设计目标

IF 的问卷不应只是心理测试，也不应只给玩家贴一个固定人格标签。问卷的真正用途是：

1. 生成玩家或角色的初始关系倾向；
2. 为事件权重、NPC反应、谎言破绽、信任变化和长期记忆提供输入；
3. 记录“自述人格”和“真实行为人格”的差异；
4. 支持不同玩家的时间投入差异；
5. 为超级真实版玩家提供完整人格/关系建档体验。

问卷输出不用于医学、心理诊断或现实人物评价，只用于游戏内概率性行为模拟。

## 2. 问卷模式分档

后续问卷建议分为四档。

| 模式 | 题量 | 面向玩家 | 目标 | 是否默认 |
| --- | ---: | --- | --- | --- |
| 快速版 | 约25题 | 想快速开始游戏的玩家 | 得到粗略画像 | 否 |
| 标准版 | 约45题 | 普通玩家 | 平衡准确度和时间 | 是 |
| 深度版 | 约80题 | 愿意认真代入的玩家 | 较高准确度 | 否 |
| 超级真实版 | 100-180题，全题库 | 不在乎时间、追求真实模拟的玩家 | 完整人格/关系档案 | 否 |

### 2.1 超级真实版要求

超级真实版不能只是“更多题”。它应该提供完整建档体验。

必须支持：

- 中途保存；
- 继续填写；
- 显示当前进度，例如 `47 / 150`；
- 显示当前模块，例如“诚实与隐瞒”；
- 允许跳过某个模块，但降低结果可信度；
- 允许从超级真实版降级为深度版；
- 最终报告显示调查完整度和结果可信度。

超级真实版适合不急着进入剧情、只想追求关系模拟真实度的玩家。

## 3. 题库模块结构

完整题库建议按模块组织。题库总量可以先以 150 题为目标，后续继续扩展。

| 模块 | 目标题量 | 主要用途 |
| --- | ---: | --- |
| 基础资料 | 8 | 年龄、性别表达、取向、关系入口、生活状态 |
| 恋爱经历 | 8 | 前任、分手原因、创伤、长期关系经验 |
| 依恋与亲密 | 12 | 怕被抛弃、怕亲密、需要陪伴、需要空间 |
| 信任与怀疑 | 12 | 查证冲动、解释接受度、旧账记忆 |
| 占有欲与边界 | 12 | 异性社交、隐私、公开关系、排他需求 |
| 沟通与冲突 | 12 | 冷战、爆发、道歉、复盘、逃避 |
| 诚实与隐瞒 | 14 | 撒谎、半真半假、秘密管理、被揭穿反应 |
| 欲望与忠诚 | 12 | 新鲜感、暧昧、诱惑抵抗、关系替代 |
| 权力与自尊 | 10 | 操控、讨好、优越感、被崇拜需求 |
| 道德与责任 | 10 | 承诺、后果、补偿、自我合理化 |
| 社交网络 | 10 | 朋友、前任、共同圈子、流言传播 |
| 现实压力 | 10 | 金钱、时间、距离、家庭、工作学习 |
| 数字生活 | 8 | 手机、社交软件、小号、定位、聊天记录 |
| 双标与共情 | 6 | 自己能不能做、对方能不能做 |
| 反向验证 | 6 | 检测自我美化、前后矛盾、乱选 |

总计约 150 题。

## 4. 题型系统

问卷不能只使用单选题。真实关系中的许多反应不是单一答案，而是“实际做法 + 同时存在的念头 + 原因 + 确信度”。

### 4.1 推荐题型

| 题型 | 用途 |
| --- | --- |
| 强制单选 | 用于现实中必须立刻做出一个行为的场景 |
| 可多选 | 用于动机、担忧、底线、原因、触发点 |
| 主选项 + 次选项 | 用于玩家主要行为和次要冲动并存的情况 |
| 多选并排序 | 用于价值优先级、最不能接受事项 |
| 多选并分配权重 | 用于多个原因同时成立的情况 |
| 滑条题 | 用于程度测量，例如嫉妒、信任、回避 |
| 二维点击题 | 用于连续坐标，例如依恋、边界、诚实 |
| 情境选择题 | 用于预测真实行为倾向 |
| NPC视角题 | 用于检测双标、共情和投射 |
| 反向验证题 | 用于检测前后矛盾和自我美化 |
| 开放文本题 | 少量使用，用于玩家补充经历或解释 |

### 4.2 单选题的弹性结构

有些题本质上需要玩家做一个主要行为，但玩家可能同时存在第二反应。建议数据结构保留弹性。

```text
question_id: Q_HONESTY_001
question_type: primary_with_secondary
primary_choice: vague_explanation
secondary_impulse: tell_truth
reason_tags:
  - fear_conflict
  - think_unnecessary_to_explain
confidence: 65
```

字段说明：

| 字段 | 含义 |
| --- | --- |
| `primary_choice` | 玩家实际会做出的行为 |
| `secondary_impulse` | 玩家同时存在但没有执行的念头 |
| `reason_tags` | 玩家选择该行为的原因，可以多选 |
| `confidence` | 玩家对该选择的确定程度，0-100 |

示例题：

> 你和异性朋友单独吃饭，恋人突然来电话问你在哪里。你会怎么做？

可记录为：

- 实际行为：说在外面吃饭，但不说和谁；
- 次要念头：其实想直接说实话；
- 原因：怕对方生气、觉得没必要解释太多；
- 确信度：65%。

这种结构比硬性单选更接近真实心理。

## 5. 二维点击题方向

二维点击题适合测连续人格坐标，不适合替代所有题目。建议用于以下坐标。

| 坐标 | 横轴 | 纵轴 | 主要影响维度 |
| --- | --- | --- | --- |
| 依恋坐标 | 不怕被抛弃 ←→ 很怕被抛弃 | 愿意亲密 ←→ 害怕亲密 | 忧虑被弃、回避亲密、安全感、关系依赖 |
| 信任坐标 | 默认相信 ←→ 默认怀疑 | 不查证 ←→ 强烈查证 | 基础信任、怀疑敏感、查证冲动 |
| 边界坐标 | 关系自由 ←→ 高度排他 | 尊重隐私 ←→ 强控制 | 占有欲、控制欲、边界感、开放接受度 |
| 诚实坐标 | 完全坦白 ←→ 选择性隐瞒 | 不会撒谎 ←→ 擅长圆谎 | 诚实、隐瞒、撒谎能力、秘密管理 |
| 冲突坐标 | 直接沟通 ←→ 回避冲突 | 情绪稳定 ←→ 情绪爆发 | 直接沟通、冷处理、爆发、防御性 |
| 欲望坐标 | 稳定长期 ←→ 新鲜刺激 | 克制冲动 ←→ 容易被诱惑带走 | 新鲜感、忠诚、诱惑抵抗、关系替代 |
| 权力坐标 | 平等关系 ←→ 想占上位 | 真诚连接 ←→ 策略操控 | 操控、优越感、被崇拜需求、权力交换 |
| 责任坐标 | 重视承诺 ←→ 重视当下 | 承担后果 ←→ 逃避后果 | 责任、承诺、后果预判、自我合理化 |

## 6. 底层维度扩展方向

当前 `docs/design/02_personality_questionnaire.md` 已有安全感、占有欲、冲突回避、表达直接度、金钱敏感度、撒谎倾向、双重标准倾向、边界敏感度、查证倾向、责任感等维度。后续完整问卷以 `docs/design/16_questionnaire_dimension_table.md` 的 **128维主表** 为准。

### 6.1 维度启用策略

128维是完整设计框架，不代表第一版代码必须全部启用。建议分阶段：

| 阶段 | 启用范围 | 目标 |
| --- | --- | --- |
| 核心启用 | 36-72个核心维度 | 先支撑问卷、事件权重和报告主标签 |
| 完整启用 | 128维主表 | 支撑超级真实版、NPC四层档案、行为人格修正 |
| 扩展预留 | 160维 | 后续增加工作/学业、身体边界、文化身份、长期成长等扩展层 |

因此后续文档以 **128维主表 + 36-72核心启用 + 160扩展预留** 为统一口径。

### 6.2 16个维度大类

| 大类编号 | 大类 | 作用 |
| --- | --- | --- |
| A01 | 基础人格与气质 | 角色的基本反应风格 |
| A02 | 自我认知与自尊 | 自我评价、面子、认可需求 |
| A03 | 情绪调节与压力反应 | 情绪波动、压力、恢复能力 |
| A04 | 社交方式 | 社交主动性、边界、孤独耐受 |
| A05 | 亲密关系与依恋 | 亲密、依赖、独立、被弃焦虑 |
| A06 | 信任、怀疑与安全感 | 信任建立、查证、旧账、投射 |
| A07 | 诚实、隐瞒与信息管理 | 坦白、撒谎、半真半假、秘密系统 |
| A08 | 冲突与沟通 | 争吵、冷处理、防御、修复 |
| A09 | 控制、边界与权力感 | 占有、控制、双标、操控 |
| A10 | 欲望、新鲜感与忠诚 | 稳定、新鲜、诱惑、替代倾向 |
| A11 | 道德、责任与后果意识 | 承诺、补偿、底线、自我合理化 |
| A12 | 价值观与人生取向 | 成就、自由、归属、精神共鸣 |
| A13 | 现实生活稳定性 | 时间、金钱、作息、适应能力 |
| A14 | 数字生活与社交媒体 | 手机、账号、定位、社交展示 |
| A15 | 家庭、成长与关系脚本 | 家庭影响、成长经验、关系模板 |
| A16 | 风险行为与危机模式 | 极端压力、破罐破摔、危机升级 |

## 7. 调查结果结构

调查结束后不应只输出一个“人格类型”。建议输出以下结构。

```text
questionnaire_result:
  mode: super_realistic
  completed_questions: 150
  total_questions: 150
  completeness: high
  reliability: high
  primary_profile: fearfully_attached_pull_push
  secondary_tags:
    - high_suspicion_sensitivity
    - pressure_based_concealment
    - strong_need_for_confirmation
  risk_tags:
    - high_checking_impulse
    - cold_war_risk
    - difficult_trust_repair
  consistency_score: medium
  self_beautification_risk: medium_high
  double_standard_risk: high
```

### 7.1 报告内容

玩家可读报告建议包含：

- 调查完整度；
- 结果可信度；
- 主画像；
- 副标签；
- 高风险关系场景；
- 明显矛盾；
- 适合的关系模式；
- 不适合的关系模式；
- 游戏内影响；
- 后续行为修正提示。

报告不应直接使用诊断式表达，例如“你就是某种病态人格”。应使用行为层和关系层语言，例如：

> 你在自己面对异性社交时更强调自由，但在对方面对异性社交时更强调解释和排他。系统判断：存在中高双标倾向。

## 8. 自述人格与行为人格

开局问卷只能得到玩家自述或角色自我认知。游戏运行过程中必须继续记录真实行为。

建议拆成：

| 档案 | 含义 |
| --- | --- |
| `self_report_profile` | 开局问卷得到的自述人格 |
| `behavior_profile` | 游戏中实际行为形成的行为人格 |
| `profile_gap` | 两者差异 |

如果差异过大，可触发标签：

- 自我美化；
- 表里不一；
- 关系伪装；
- 自我欺骗；
- 双标倾向。

示例：

```text
self_report_profile:
  honesty_tendency: high
  boundary_respect: high

behavior_profile:
  concealment_tendency: high
  checking_impulse: high

profile_gap:
  self_beautification: high
  double_standard_risk: medium_high
```

## 9. NPC四层档案

每个重要 NPC 不应只有真实人格。为了支持伪装、误判、暧昧、谎言和双重身份，应保留四层档案。

| 档案 | 含义 | 玩家是否可见 |
| --- | --- | --- |
| `true_profile` | NPC真实人格，系统知道 | 不可见 |
| `self_report` | NPC自己怎么描述自己，可能不准 | 部分可见 |
| `shown_profile` | NPC表现给外界的人设，可能伪装 | 可见 |
| `player_perception` | 玩家当前以为对方是什么样，可能错误 | 可见 |

支持的剧情：

- 表面温柔稳定，真实高隐瞒；
- 自称不爱撒谎，但压力下经常半真半假；
- 对外人设专一，实际有小号或暧昧对象；
- 自己可以和异性吃饭，但不允许玩家做同样事情。

## 10. 谎言与证据系统

撒谎不应只是数值加减。每个谎言应记录信息链。

```text
lie_record:
  lie_id: LIE_SOC_001
  truth: 和异性朋友单独吃饭
  statement: 和朋友吃饭
  omitted_parts:
    - 单独
    - 异性
  told_to: player
  time: day_4_evening
  evidence:
    - payment_record
    - friend_photo
  witnesses:
    - common_friend_A
  contradiction_risk: medium
  exposure_risk: medium_high
  explanation_room: medium
```

该系统用于产生：

- 圆谎压力；
- 前后矛盾；
- 朋友撞见；
- 截图或照片证据；
- 付款记录；
- 被揭穿后的解释空间；
- NPC或玩家的长期记忆伤口。

## 11. 多层信任系统

不要只使用一个总信任值。建议拆成多层。

| 信任类型 | 含义 |
| --- | --- |
| 情感信任 | 相信对方还在乎自己 |
| 忠诚信任 | 相信对方不会暧昧或背叛 |
| 事实信任 | 相信对方说的话是真的 |
| 行动信任 | 相信对方答应的事会做到 |
| 隐私信任 | 相信对方不会偷看、查手机或侵犯边界 |
| 危机信任 | 相信出事时对方不会逃避 |

这样可以表达更细的关系状态，例如：

- 相信你爱我，但不相信你不会撒谎；
- 相信你不会出轨，但不相信你会守约；
- 相信你平时温柔，但不相信你冲突时会负责。

## 12. 长期记忆与情绪分层

情绪和关系不应只做即时数值变化。

建议分层：

| 层级 | 含义 |
| --- | --- |
| 即时情绪 | 当前开心、生气、委屈、失望 |
| 短期状态 | 这几天冷淡、黏人、怀疑、疲惫 |
| 长期印象 | 对方是否靠谱、是否尊重边界 |
| 关系伤口 | 被欺骗、被羞辱、被冷落等长期记忆 |
| 修复进度 | 道歉、解释、补偿是否逐步恢复 |

同一行为在不同记忆背景下应产生不同反应。例如第一次迟到只是小不满，第三次迟到可能触发“长期不被重视”的伤口。

## 13. 社交网络与现实压力

真实关系不是两个人封闭互动。后续应逐步引入：

- 朋友；
- 前任；
- 同学或同事；
- 家庭；
- 共同好友；
- 暧昧对象；
- 流言传播；
- 截图、照片、共同圈子证据；
- 时间压力；
- 金钱压力；
- 距离压力；
- 学业或工作压力；
- 作息不一致。

真实行为的形成应接近：

```text
倾向 + 机会 + 压力 + 诱因 + 风险判断 = 行为概率
```

例如高新鲜感不必然出轨，如果社交圈小、责任感高、风险高，可能只是心动不行动；中等新鲜感在长期冷淡、有人主动安慰、解释空间大的情况下，反而可能进入暧昧。

## 14. 分阶段实施建议

后续不要一次性实现全部复杂系统。建议按以下顺序推进。

### 阶段 A：文档与题库设计

- 创建完整问卷升级文档；
- 定义题型结构；
- 定义 128维主表，并允许第一版只启用36-72个核心维度；
- 写 150 题超级真实版题库；
- 写报告模板；
- 写 NPC 四层档案样例；
- 写谎言证据链样例。

### 阶段 B：最小控制台问卷原型

- 支持快速版、标准版、深度版；
- 超级真实版可以先只做全题库顺序填写；
- 先不做 UI，只做控制台；
- 输出玩家可读报告。

### 阶段 C：自述人格与行为人格

- 开局生成 `self_report_profile`；
- 游戏中记录 `behavior_profile`；
- 结算时对比差异；
- 输出自我美化、双标、表里不一等标签。

### 阶段 D：NPC四层档案与多层信任

- NPC 增加真实、自述、外显、玩家认知四层档案；
- 信任拆为情感、忠诚、事实、行动、隐私、危机；
- 事件根据不同信任层产生不同后果。

### 阶段 E：谎言证据链与长期记忆

- 记录每个谎言的事实、说法、隐瞒点和证据；
- 支持被撞见、截图、朋友传话等暴露方式；
- 支持长期关系伤口和修复进度。

### 阶段 F：社交网络与现实压力

- 引入朋友、前任、共同好友、家庭、工作学习压力；
- 让事件不只来自两人对话，也来自外部网络和现实条件。

## 15. 非目标

本升级方案不是要立刻把 IF 变成完整大型游戏。

当前不做：

- UI；
- AI API；
- 医学或心理诊断；
- 成人露骨内容；
- 完整一年模拟；
- 所有复杂系统一次性接入；
- 通过题目直接判断现实玩家人格好坏。

当前重点是把真实关系模拟所需的数据结构、问卷结构和报告逻辑先沉淀下来。
