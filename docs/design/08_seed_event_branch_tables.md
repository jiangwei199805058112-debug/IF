# 种子事件分支表 v0.1

本文件把 `05_seed_events.md` 中的 3 个种子事件拆成更可测试的分支表。分支表用于后续纸面测试和最小实现，不新增大量事件。

## 1. 设计目标

- 把说明型事件转成可验证的分支结构。
- 明确每个分支的真实情况、玩家看到的信息、NPC 解释方式和后续钩子。
- 保留真实层与感知层差异。
- 使用行为标签和方向性后果，不写固定数值。
- 为 14 天流程提供必触发事件的测试素材。

## 2. 分支表字段说明

| 字段 | 说明 |
| --- | --- |
| 分支ID | 事件内唯一，例如 `MSG_001_A` |
| 真实情况 | 系统内部真实发生的事 |
| 玩家看到的信息 | 玩家可见内容，不包含隐藏事实 |
| NPC解释方式 | 坦白、敷衍、半真半假、隐瞒、模糊、反向指责等 |
| 玩家可选行为 | 玩家可执行选择 |
| 主要行为标签 | 本分支涉及的核心标签 |
| 真实后果 | 内部真实值和状态方向变化 |
| 感知反馈 | 玩家可见的模糊反馈 |
| 破绽变化 | 无、低幅上升、中幅上升、明显上升等 |
| 记忆写入 | 不写入、条件写入、必定写入 |
| 后续钩子 | 后续可能触发的事件或状态 |

## 7. v0.1 测试重点

v0.1 测试时重点观察：

- 玩家是否能接受“看不到真实值，只看模糊反馈”。
- 同一事件不同真实分支是否会带来不同判断。
- 半真半假和隐瞒重点是否比完全撒谎更常用、更可测试。
- 破绽值是否能自然引出查证和暴露。
- 平淡日是否能修复轻微不安。
- 冷战、拉黑和公共场合争吵是否能进入记忆账本。
- 第 14 天是否能根据这些分支输出不同关系结果。

仍然不要写死数值。分支后果使用：

- 真实信任小幅下降。
- 感知安全感明显下降。
- 破绽值中幅上升。
- 未解决冲突写入记忆。
- 分手危机权重上升。
