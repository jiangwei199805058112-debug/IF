# 语言沟通、自我表露与秘密边界系统设计

本文档用于沉淀 IF 项目中的语言沟通、自我表露、回应性、秘密与禁忌话题机制。它不是普通沟通建议，而是用于支撑问卷建档、NPC 互动、关系事件、信任变化、旧伤记忆和关系报告的系统设计。

核心原则：

```text
亲密关系不是只靠“好感”推进，而是靠双方如何交换真实信息、如何回应脆弱、如何处理秘密、如何谈论禁忌话题来逐步形成安全感和亲密感。
```

本设计参考亲密关系中的语言沟通、自我表露、社会渗透、回应性、隐私边界、秘密管理和依恋沟通差异等观点，并转译为 IF 的系统机制。本文档不收录原始截图或长篇原文，只保留可供项目使用的变量、事件模板、报告标签和落地边界。

---

## 1. 设计目的

IF 当前已经具备：

- 问卷 MVP；
- 真实原因层与解释可信度系统；
- 伴侣认知准确度、形象管理与危险知觉系统；
- `relationship_interpretation.py` 关系事件解释原型。

本系统要补足另一个核心问题：

```text
亲密关系中，两个人到底是如何通过语言、表露、倾听、回应和保留秘密来靠近或疏远的？
```

IF 不应把沟通写成固定的“多说话就加分”。更合理的是：

```text
沟通质量 = 表露内容 + 表露深度 + 表露节奏 + 对方回应性 + 互惠程度 + 隐私边界匹配度
```

其中任何一项失衡，都可能导致关系变化：

- 说得太少：距离感、猜测和不安全感上升；
- 说得太快：压力、尴尬和逃避上升；
- 只一方表露：互惠失衡和不公平感上升；
- 对方回应敷衍：受伤、羞耻和后续沉默上升；
- 秘密边界不匹配：信任危机和控制感冲突上升。

## 2. 系统一：自我表露系统

### 2.1 设计原则

自我表露是亲密关系发展的重要指标，但不是越多越好，也不是越早越好。IF 中的自我表露应同时考虑：

```text
说什么？
说多深？
什么时候说？
对谁说？
对方如何回应？
这次表露是否换来了更安全的关系？
```

### 2.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `self_disclosure_willingness` | 主动透露个人信息、真实想法和脆弱感的意愿 |
| `self_disclosure_breadth` | 表露话题广度 |
| `self_disclosure_depth` | 表露内容深度 |
| `disclosure_pacing` | 表露节奏，过快或过慢都可能带来关系成本 |
| `over_disclosure_risk` | 关系还浅时过度表露造成的压力风险 |
| `under_disclosure_gap` | 长期不表露造成的距离感 |
| `vulnerability_trust_gain` | 适度脆弱表露带来的信任增长 |
| `pressure_disclosure_pattern` | 只有争吵、崩溃或被逼问时才表露的模式 |
| `selective_disclosure` | 只选择性说出对自己有利或低风险的信息 |
| `disclosure_regret` | 表露后后悔、羞耻或担心被利用的程度 |

### 2.3 表露层级

自我表露可以分为三层：

| 层级 | 内容例子 | 游戏用途 |
| --- | --- | --- |
| 浅层 | 日常、兴趣、吃饭、工作学习、轻松玩笑 | 建立基础熟悉度 |
| 中层 | 家庭、前任、价值观、未来计划、压力来源 | 建立关系理解和信任 |
| 深层 | 创伤、自卑、欲望、边界、金钱压力、做错过的事、秘密 | 建立深度亲密，也可能触发压力和风险 |

系统不应让深层话题无条件正收益。深层表露需要结合关系阶段、对方回应性、信任水平和表露节奏判断。

### 2.4 事件模板

事件：你向对方说起一件过去让你很难受的事。

可能结果：

1. 对方认真听、追问细节、表达理解：安全感和亲密感上升；
2. 对方安慰但很快转移话题：轻微失落，表露意愿小幅下降；
3. 对方评价你“想太多”：羞耻感和旧伤记忆上升；
4. 对方拿你的脆弱开玩笑：信任明显下降；
5. 对方沉默不回应：玩家进入不确定解释；
6. 对方也分享自己的类似经历：互惠表露上升，亲密感加深。

## 3. 系统二：社会渗透、话题广度与深度

### 3.1 设计原则

亲密关系通常不是一步进入深层，而是从少量浅层话题逐渐扩展到更多、更深、更有个人意义的话题。

IF 可以将关系推进理解为：

```text
话题广度扩大 + 话题深度加深 + 互惠回应稳定 → 亲密感逐渐建立
```

### 3.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `topic_breadth` | 关系中可谈话题的数量和类型 |
| `topic_depth` | 话题触及个人意义、情绪和隐私的深度 |
| `topic_layer_progression` | 从浅层到深层的推进是否自然 |
| `deep_topic_readiness` | 当前关系是否已经准备好谈深层话题 |
| `topic_jump_risk` | 过快跳到深层话题带来的压力 |
| `safe_topic_zone` | 双方都能自然谈论的话题范围 |
| `blocked_topic_zone` | 一方或双方长期回避的话题范围 |

### 3.3 话题层级例子

浅层话题：

- 今天吃什么；
- 工作/学习日常；
- 兴趣爱好；
- 轻松吐槽；
- 共同娱乐。

中层话题：

- 家庭关系；
- 前任经历；
- 价值观；
- 未来计划；
- 对恋爱的期待；
- 对承诺的理解。

深层话题：

- 创伤；
- 自卑；
- 欲望和性边界；
- 金钱压力；
- 原生家庭痛点；
- 隐瞒与秘密；
- 关系底线；
- 曾经做错的事。

## 4. 系统三：回应性系统

### 4.1 设计原则

亲密感不只是“我说了什么”，更取决于“对方如何回应”。有意义的表露如果没有被接住，可能比不表露更伤人。

IF 中的回应性应包括：

```text
理解、关心、兴趣、尊重、情绪确认、适度追问、不急着评价。
```

### 4.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `perceived_responsiveness` | 感受到对方有回应的程度 |
| `empathic_response_quality` | 共情回应质量 |
| `emotional_validation` | 情绪是否被承认和接住 |
| `dismissive_response_risk` | 敷衍、打断、转移话题、嘲笑或否定的风险 |
| `safe_disclosure_space` | 是否感觉可以安心说真话 |
| `curiosity_about_partner` | 是否主动了解对方 |
| `listening_quality` | 倾听质量 |
| `problem_solving_bias` | 过早给方案、忽略情绪的倾向 |
| `judgmental_response` | 评价、审判或道德压迫式回应 |

### 4.3 事件模板

事件：玩家说“最近我其实有点撑不住”。

NPC 可能回应：

| 回应 | 效果 |
| --- | --- |
| “你愿意说给我听吗？我在。” | 安全感和亲密感上升 |
| “那你打算怎么解决？” | 对问题解决型玩家可能有帮助，对情绪确认需求高者可能失落 |
| “大家都很累，别想太多。” | 表露意愿下降 |
| “你怎么这么脆弱？” | 信任明显下降，形成旧伤记忆 |
| 转移话题 | 玩家可能解释为不关心或不会回应 |

## 5. 系统四：秘密、隐私与禁忌话题

### 5.1 设计原则

亲密关系不等于没有隐私。保留秘密不一定错误，但重要秘密、长期隐瞒、被发现后的解释，会严重影响信任。

IF 需要区分：

```text
正当隐私 ≠ 伤害性隐瞒 ≠ 欺骗 ≠ 背叛。
```

### 5.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `privacy_need` | 正当隐私需求 |
| `privacy_boundary_strength` | 隐私边界强度 |
| `secret_burden` | 秘密负担 |
| `secret_moral_weight` | 秘密的道德重量和关系伤害潜力 |
| `taboo_topic_avoidance` | 回避禁忌话题的倾向 |
| `avoidance_cost` | 长期回避讨论带来的关系成本 |
| `disclosure_timing` | 坦白时机是否合适 |
| `forced_disclosure_damage` | 被逼问或被发现后才说造成的伤害 |
| `secret_discovery_shock` | 秘密被发现后的冲击 |
| `privacy_boundary_respect` | 是否尊重彼此隐私边界 |

### 5.3 事件模板

事件：对方不愿谈前任。

真实原因可能是：

1. 这是对方的隐私边界；
2. 对方还没准备好；
3. 对方担心你比较或受伤；
4. 对方确实还有未处理的联系；
5. 对方隐瞒了会影响关系的重要事实；
6. 对方习惯回避所有敏感话题。

玩家解释可能是：

1. “每个人都有隐私。”
2. “他/她还没放下。”
3. “是不是有事瞒我？”
4. “他/她不信任我。”
5. “我是不是问太多了？”

系统应结合：

- 当前关系阶段；
- 之前是否主动说过边界；
- 是否存在可见线索；
- 是否涉及当前关系利益；
- 是否被发现后才解释；
- 玩家与 NPC 的隐私边界是否匹配。

## 6. 系统五：开启者与倾听者能力

### 6.1 设计原则

有些人更容易让别人敞开心扉，不是因为他们话多，而是因为他们提供了安全、兴趣、非评判和稳定回应。

同一句“你怎么了？”，不同人说出来可能像关心，也可能像审问。

### 6.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `opener_skill` | 让别人敞开心扉的能力 |
| `conversation_invitation` | 引导对方表达的能力 |
| `nonjudgmental_attitude` | 非评判态度 |
| `emotional_safety_signal` | 让对方感到安全的信号 |
| `attentive_listening` | 专注倾听程度 |
| `encouragement_to_share` | 鼓励对方表达感受的能力 |
| `intrusive_questioning_risk` | 追问过度导致像审问的风险 |

### 6.3 游戏表现

| 高开启者 | 低开启者 |
| --- | --- |
| 对方更愿意主动说 | 对方更容易沉默 |
| 表露后更少后悔 | 表露后更容易防御 |
| 更容易进入深层话题 | 话题停留在表层 |
| 冲突后更可能修复 | 冲突后更可能冷处理 |

## 7. 系统六：依恋类型与沟通差异

### 7.1 设计原则

依恋类型会影响沟通方式，但不能写成固定人格标签。IF 应把依恋作为倾向，而不是诊断。

### 7.2 沟通差异

| 依恋倾向 | 沟通表现 |
| --- | --- |
| 安全倾向 | 更愿意表达、倾听、回应，也更能尊重隐私边界 |
| 焦虑倾向 | 更需要确认，更容易追问、过度表露、害怕冷落 |
| 回避倾向 | 更少表露，更重视隐私，更容易回避深层话题 |
| 恐惧倾向 | 想亲近但害怕受伤，可能忽冷忽热或表露后撤退 |

### 7.3 事件模板

事件：一方说“我想知道你以前为什么分手”。

- 安全倾向：可能愿意说大概，但保留细节；
- 焦虑倾向：可能追问细节，用于确认风险；
- 回避倾向：可能觉得被侵犯，迅速转移话题；
- 恐惧倾向：可能先说一些，又突然后悔或退缩。

## 8. 系统七：爱意表达与关系稳定

### 8.1 设计原则

亲密关系中的语言表达不只是信息交换，也包括直接表达喜欢、关心、感谢、承诺和保护。适当表达爱意能增强安全感，但空洞承诺或只说不做会损害可信度。

### 8.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `affection_expression_frequency` | 爱意表达频率 |
| `affection_expression_clarity` | 爱意表达清晰度 |
| `warmth_in_language` | 语言中的温度 |
| `verbal_commitment_credibility` | 口头承诺可信度 |
| `words_actions_consistency` | 语言与行动一致性 |
| `under_expression_hurt` | 表达不足造成的受伤 |
| `empty_promise_risk` | 空洞承诺风险 |

### 8.3 事件模板

事件：玩家说“我爱你”，但对方沉默或转移话题。

可能原因：

1. 对方还没准备好；
2. 对方不习惯直接表达；
3. 对方确实投入不足；
4. 对方害怕承诺；
5. 对方觉得行动比语言重要；
6. 双方对语言表达的需求不匹配。

## 9. 报告标签设计

### 9.1 标签边界原则

报告标签不能把低表露者写成“冷漠”，也不能把高表露者写成“情绪化”。标签应描述沟通模式、适配条件和风险点。

### 9.2 可用标签

| 标签 ID | 含义 | 报告表达方向 |
| --- | --- | --- |
| `deep_discloser` | 深层表露者 | 更愿意分享深层经历和脆弱点，适合回应性高的伴侣 |
| `guarded_discloser` | 谨慎表露者 | 需要较高安全感才会说真实想法 |
| `fast_disclosure_pattern` | 快速表露模式 | 容易在关系早期说很多，可能拉近关系，也可能给对方压力 |
| `reciprocity_sensitive` | 互惠敏感 | 在自己说了很多而对方很少回应时容易失衡 |
| `privacy_protective` | 隐私保护强 | 重视个人边界，不适合被强行追问 |
| `taboo_avoider` | 禁忌话题回避 | 容易绕开前任、金钱、家庭、欲望、承诺等敏感话题 |
| `high_responsiveness_need` | 高回应需求 | 说出脆弱内容后需要明确理解和安抚 |
| `opener_type` | 开启者 | 容易让别人放松、表达和分享 |
| `low_verbal_affection` | 低语言爱意表达 | 不一定不爱，但可能让高表达需求伴侣不安 |
| `words_actions_gap` | 言行落差 | 口头表达与实际行为可能不一致，需要后续行为校验 |

## 10. 与前期问卷的关系

本系统非常适合进入前期调查，但不应只问“你是否愿意表露”。应拆成：

```text
你通常说多少？
你说到多深？
你什么时候说？
你希望对方怎么回应？
你希望对方也说多少？
哪些事你认为可以保留？
哪些事你认为必须坦白？
```

建议对应问卷模块见：

```text
docs/design/28_questionnaire_communication_disclosure_module.md
```

## 12. 不做什么

本系统暂不做：

1. 不把高表露等同于成熟；
2. 不把低表露等同于冷漠；
3. 不把保留隐私等同于欺骗；
4. 不把追问细节等同于关心；
5. 不把“说我爱你”直接等同于高忠诚；
6. 不把一次沉默直接判定为不爱；
7. 不写成固定加分或扣分系统；
8. 不做心理诊断；
9. 不接 AI API；
10. 不上传原始截图或大段原文。

## 13. 一句话总结

```text
IF 的语言沟通系统要模拟的不是“说得多不多”，而是：
两个人如何逐步交换真实信息，如何接住彼此脆弱，如何处理隐私和秘密，以及这些沟通习惯如何让关系靠近或疏远。
```
