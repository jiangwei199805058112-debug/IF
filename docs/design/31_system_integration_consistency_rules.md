# IF 关系系统整合一致性与冲突修正规则

本文档用于对 `25` 到 `30` 号设计文档进行横向校对，明确各系统之间的边界、优先级、数据流和冲突修正口径。它不是新增独立玩法，而是用于防止后续实现时出现重复扣分、逻辑互相打架、同一行为被多个系统重复解释或错误叠加。

覆盖文档：

```text
docs/design/25_attribution_memory_belief_system.md
docs/design/26_partner_perception_and_impression_system.md
docs/design/27_communication_self_disclosure_system.md
docs/design/28_questionnaire_communication_disclosure_module.md
docs/design/29_conflict_communication_repair_system.md
docs/design/30_social_exchange_dependency_system.md
```

---

## 1. 总体结论

经横向整理，当前新增系统与原有系统没有根本矛盾，但存在若干后续实现时容易出问题的点：

1. “自我表露越多越亲密”容易和“隐私边界”冲突；
2. “沉默/不回应”容易被误写成一律冷暴力或石墙；
3. “争吵”容易被误写成一律扣分；
4. “满意度”容易被误写成“稳定性”；
5. “社会交换”容易被误写成冷冰冰的算计；
6. “怀疑”容易被误写成焦虑，“信任”容易被误写成成熟；
7. “较少依赖的一方更有权力”容易被误写成一定更坏或一定操控；
8. 多个系统可能对同一事件重复扣信任、扣满意度或写旧伤记忆。

因此后续实现必须采用统一数据流和统一裁决规则。

---

## 2. 统一事件处理顺序

所有关系事件建议按以下顺序处理：

```text
1. 真实原因层 truth layer
2. 可见线索层 observable traces / evidence
3. 角色基础画像 questionnaire profile
4. 伴侣认知层 partner perception
5. 归因解释层 attribution / interpretation
6. 沟通行为层 communication action
7. 回应性与修复层 responsiveness / repair
8. 奖赏代价层 rewards / costs
9. 满意度与依赖层 satisfaction / dependence
10. 记忆与关系信念更新 memory / belief update
11. 报告标签 report tags
```

关键原则：

```text
先判断真实发生了什么，
再判断角色看见了什么，
再判断角色如何理解，
再判断角色如何表达和回应，
最后才结算信任、满意度、依赖度和长期记忆。
```

不要让某一个系统直接越级决定最终关系结局。

## 3. 事实层、解释层、认知层的边界

### 3.1 三者不应混用

| 层级 | 回答的问题 | 对应文档 |
| --- | --- | --- |
| 真实原因层 | 事情真实是什么？对方真实意图是什么？ | 25 |
| 解释可信度层 | 对方给出的解释是否可信？线索是否支持？ | 25 |
| 伴侣认知层 | 玩家/NPC 为什么这样理解对方？这种理解准不准？ | 26 |

示例：

```text
事实：对方确实看到了消息，但没有回。
解释：对方说“我没看手机”。
线索：期间发了朋友圈。
认知：玩家认为“对方对别人可用，对我不可用”。
归因：玩家进一步解释为“不重视我”或“故意冷处理”。
```

这四层不能压缩成一个简单扣分。

### 3.2 怀疑与信任必须保留四象限

| 判断 | 事实 | 类型 |
| --- | --- | --- |
| 怀疑有问题 | 真的有问题 | 准确警觉 |
| 怀疑有问题 | 实际没问题 | 误会/焦虑 |
| 相信没问题 | 实际没问题 | 稳定信任 |
| 相信没问题 | 真的有问题 | 被欺骗/低估风险 |

后续实现中禁止写成：

```text
怀疑 = 错误
信任 = 正确
```

也禁止写成：

```text
怀疑 = 一定准确
信任 = 一定愚蠢
```

## 4. 自我表露与隐私边界的统一口径

### 4.1 表露不是越多越好

`27` 文档强调自我表露能推动亲密，但 `27` 和 `28` 也都明确：表露需要节奏、边界和回应性。

统一规则：

```text
亲密收益 = 适度表露 + 合适时机 + 对方回应性 + 互惠程度
```

而不是：

```text
表露越多 → 关系越好
```

### 4.2 隐私不是欺骗

后续实现必须区分：

| 类型 | 含义 | 处理方式 |
| --- | --- | --- |
| 正当隐私 | 不影响当前关系安全和承诺的信息 | 尊重边界，不直接扣信任 |
| 边界敏感信息 | 可能影响伴侣感受，但未必构成伤害 | 进入协商和解释可信度判断 |
| 伤害性隐瞒 | 与当前关系权益、忠诚或安全有关且长期不说 | 扣事实信任，写入旧伤记忆 |
| 欺骗/背叛 | 主动撒谎、掩盖、转移证据或违背承诺 | 强烈影响忠诚信任和关系结果 |

因此：

```text
保留隐私 ≠ 撒谎
不愿谈某事 ≠ 一定有问题
被发现后解释前后不一致 ≠ 普通隐私
```

### 4.3 表露互惠不能写成强制透明

如果玩家希望对方表露更多，应判断这是：

1. 真实的亲密需求；
2. 互惠公平需求；
3. 安全感不足；
4. 控制欲；
5. 双标式透明要求。

不要让 `partner_disclosure_expectation` 自动等于 `control_tendency`。

---

## 5. 沉默、不回应、暂停与石墙的区别

`27` 文档中，表露后对方沉默可能降低安全感；`29` 文档中，石墙会伤害关系。但两者不能混为一谈。

统一规则：

| 行为 | 条件 | 类型 |
| --- | --- | --- |
| 支持性沉默 | 对方在场、关注、不打断，允许情绪存在 | 支持 |
| 处理性沉默 | 对方需要时间组织语言，但之后回应 | 中性或轻微正向 |
| 压力性撤退 | 对方被情绪淹没，需要暂停，并说明会回来 | 可修复 |
| 回避性沉默 | 对方长期不回应、不解释、不回到问题 | 伤害性回避 |
| 石墙 | 对方拒绝回应、冷处理、用沉默惩罚或控制 | 高伤害 |

因此后续实现禁止：

```text
沉默 = 石墙
暂停 = 冷暴力
不立即回应 = 不在乎
```

有效暂停必须满足：

1. 说明需要冷静；
2. 说明不是逃避；
3. 给出回来谈的时间或条件；
4. 之后真的回来处理问题。

---

## 6. 争吵、冲突与修复的统一口径

`29` 文档已经明确：争吵不应一律扣分。

统一规则：

| 冲突类型 | 结果倾向 |
| --- | --- |
| 精确表达 + 积极倾听 + 确认感受 + 具体请求 | 可提高长期信任和修复质量 |
| 情绪激烈但能暂停并回来处理 | 短期波动，长期可能修复 |
| 偏题、旧账堆叠、读心、反向抱怨 | 冲突升级 |
| 批评人格、蔑视、羞辱、防卫、石墙 | 高伤害，写旧伤记忆 |
| 表面不吵但长期回避 | 满意度和真实亲密下降 |

因此：

```text
不吵 ≠ 关系健康
吵架 ≠ 关系失败
会修复的冲突可能比长期回避更健康
```

## 7. 满意度、信任、依赖度、稳定性的区别

这是当前最需要防止混淆的部分。

### 7.1 四个变量不能互相替代

| 概念 | 回答的问题 | 主要来源 |
| --- | --- | --- |
| 信任 | 我相信你不会伤害、欺骗或背叛我吗？ | 25、26、27、29 |
| 满意度 | 这段关系结果是否达到我的期待？ | 30 |
| 依赖度 | 我离开这段关系会损失多少？有没有更好选择？ | 30 |
| 稳定性 | 这段关系短期内是否容易结束？ | 30 + 投入 + 冲突 + 替代选择 |

高信任不必然高满意，高满意也不必然稳定。

例子：

```text
我相信你是好人，但这段关系给不了我想要的生活。
```

这是：

```text
信任不低，但满意度可能低。
```

再如：

```text
我不幸福，但分开成本太高，也没有更好的去处。
```

这是：

```text
满意度低，但依赖度高，关系可能稳定。
```

### 7.2 社会交换不是冷冰冰算计

`30` 文档中的奖赏、代价、CL、CLalt 是关系稳定性的计算框架，不应写成角色把伴侣当商品。

统一口径：

```text
社会交换系统用于模拟关系中的主观收益、主观代价、期待和替代可能性，
不是把爱情简化成金钱交易。
```

奖赏包括情绪、陪伴、尊重、共同成长、性吸引、家庭支持、现实协作等。代价包括焦虑、失望、机会成本、现实负担、自由损耗、旧伤和离开成本。

## 8. 公式口径与数值范围修正

`30` 文档中的公式是概念公式，不应在实现中直接按原始数字相加。

建议实现口径：

```text
relationship_outcome = normalized_rewards - normalized_costs
relationship_satisfaction = relationship_outcome - comparison_level
relationship_dependence = relationship_outcome - comparison_level_alternatives + investment_weight + leaving_cost_weight
```

但具体实现时必须明确：

1. 各字段统一归一化到同一范围；
2. 如果使用 0-100 分，则 50 应视为中性基线；
3. 如果使用 -100 到 100 分，则 0 可视为中性基线；
4. 不同模块的 delta 不应重复叠加到同一结果；
5. 最终结算应由一个 aggregator 汇总，而不是每个模块各自改最终关系状态。

建议后续实现新增统一聚合器概念：

```text
relationship_state_aggregator
```

用于合并：

```text
trust_delta
satisfaction_delta
intimacy_delta
dependence_delta
stability_delta
old_wound_memory_delta
repair_chance_delta
```

## 9. 重复扣分与重复写入的避免规则

同一事件可能同时触发多个系统。例如：

```text
对方说没看手机，但期间发了朋友圈，被发现后解释前后不一致，争吵中又开始防卫和反向抱怨。
```

这会触发：

- 真实原因层；
- 证据链；
- 伴侣认知；
- 归因；
- 沟通冲突；
- 奖赏代价。

实现时不能让每个系统都单独大幅扣 `trust`。

统一规则：

| 影响类型 | 主要负责系统 |
| --- | --- |
| 事实信任损失 | 25 真实原因层 / 解释可信度 |
| 认知准确度标签 | 26 伴侣认知系统 |
| 表露安全感 | 27 沟通表露系统 |
| 冲突升级和修复机会 | 29 冲突沟通系统 |
| 满意度和稳定性 | 30 社会交换系统 |
| 长期记忆写入 | 25 + 29 共同决定 |

其他系统可以提供修正因子，但不应重复主扣分。

## 10. 依赖权力系统的边界修正

`30` 文档中的“较少依赖的一方更有权力”不能被误写成：

```text
不那么依赖 = 坏人
更依赖 = 受害者
```

统一规则：

```text
依赖差异只表示关系权力结构，不自动表示道德好坏。
```

需要结合：

- 是否利用对方害怕失去来威胁；
- 是否长期让对方让步；
- 是否尊重对方边界；
- 是否存在操控、惩罚、冷暴力或经济/社交控制；
- 是否只是双方自然依赖程度不同。

只有当依赖差异与威胁、控制、羞辱、反复分手威胁等行为结合时，才进入高风险关系权力事件。

## 11. 问卷与游戏行为的优先级

前期问卷只能提供初始倾向，不能永久定义角色。

统一规则：

```text
问卷结果 = 初始画像
游戏行为 = 动态修正
多次行为证据 > 单题自述
关键事件行为 > 抽象自评
```

例如：

- 问卷说自己善于倾听，但游戏中多次打断和反向抱怨，则 `listening_quality` 应下降；
- 问卷说尊重隐私，但游戏中反复查手机，则 `privacy_boundary_respect` 应下降；
- 问卷说不在意替代选择，但游戏中频繁暧昧和比较，则 `comparison_level_alternatives` 或 `alternative_sensitive` 应上升。

## 12. 推荐实现顺序

后续 Codex 恢复后，不建议一次性把 25-30 全部接入代码。建议按低风险顺序：

1. 先补 README 索引，把 26-31 文档加入阅读顺序；
2. 在 `relationship_interpretation.py` 中补充“统一事件处理顺序”的注释或轻量结构；
3. 新增 `relationship_state_aggregator` 草案，先只汇总结果，不改主流程；
4. 增加 29 的冲突沟通测试：暂停不是石墙、争吵可修复、人格攻击高伤害；
5. 增加 30 的社会交换测试：满意度与稳定性分离；
6. 再考虑把 28 中的 4 个轻量沟通问卷题接入 MVP。

## 13. 当前需修正的文档口径

当前 25-30 文档本身没有必须立刻删除或重写的硬性矛盾，但后续阅读时应以本文档的整合规则为优先解释。

尤其是以下句子需要以后实现时按修正口径理解：

| 原口径容易误解为 | 修正口径 |
| --- | --- |
| 自我表露推动亲密 | 适度、互惠、被接住的表露推动亲密 |
| 沉默不回应会伤害关系 | 沉默需区分支持性沉默、处理性沉默、回避和石墙 |
| 争吵会带来问题 | 争吵本身不必然有害，关键是表达、倾听、确认和修复 |
| 满意度低会离开 | 满意度影响幸福感，离开还取决于 CLalt、投入和成本 |
| 较少依赖更有权力 | 权力差异是结构，不自动等于操控或道德问题 |
| 怀疑可能是焦虑 | 怀疑也可能是准确警觉，必须结合事实和证据 |

---

## 14. 一句话总结

```text
25-30 号文档不是六套互相独立的加减分系统，而是一条关系事件处理链：
事实发生 → 线索出现 → 角色理解 → 沟通表达 → 对方回应 → 冲突或修复 → 奖赏代价变化 → 满意度和依赖度变化 → 记忆与报告更新。
```
