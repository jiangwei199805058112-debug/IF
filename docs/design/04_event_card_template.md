# 事件卡模板 v0.1

事件卡用于描述 IF 中一次可模拟的关系事件。每张事件卡必须分为真实层、感知层、后果层。

原则：

- 真实层记录系统内部真实发生的事。
- 感知层记录玩家能看到、误判和选择的内容。
- 后果层记录行为标签、数值方向、记忆和后续钩子。
- 不要直接写死“好感 +5”。
- 先记录行为标签，再结合角色字段、关系阶段、压力、记忆账本推导影响。

## 1. 事件基础信息

| 字段 | 说明 |
| --- | --- |
| 事件ID | 全局唯一，例如 `MSG_001` |
| 事件名 | 简短名称，例如“对方很久没回消息” |
| 适用阶段 | 聊天、暧昧、刚恋爱等 |
| 触发条件 | 时间、压力、关系状态、前置记忆 |
| 参与角色 | 玩家、NPC、共同朋友、第三者等 |
| 是否 v0.1 启用 | 启用 / 暂不启用 |

## 2. 真实层

真实层由系统掌握，玩家不能直接看到。

| 字段 | 说明 |
| --- | --- |
| 事件ID | 与基础信息一致 |
| 真实发生了什么 | 客观事实，不加入玩家猜测 |
| 涉及角色 | 谁实际参与了事件 |
| 真实对象 | 例如同事、朋友、前任、家人、第三者候选 |
| 真实目的 | 工作、社交、情绪支持、隐瞒、试探等 |
| 是否撒谎 | 无 / 坦白 / 半真半假 / 隐瞒重点 / 完全撒谎 |
| 是否有破绽 | 无 / 低 / 中 / 高 |
| 是否存在第三者 | 无 / 潜在 / 明确 |
| 是否进入记忆账本 | 不进入 / 条件进入 / 必定进入 |
| 真实压力来源 | 工作、学校、家庭、经济、关系、身体等 |
| 真实风险 | 信任危机、冷淡、分手危机、第三者线索 |

## 3. 感知层

感知层是玩家实际看到和理解的内容。

| 字段 | 说明 |
| --- | --- |
| 玩家看到什么 | 消息、表情、朋友圈、电话、共同朋友反馈 |
| 玩家不知道什么 | 被隐瞒的对象、目的、时间线或压力来源 |
| 玩家可能误判什么 | 忙碌误判成冷淡，普通社交误判成暧昧等 |
| 可用选项 | 玩家可执行的行为标签 |
| 情绪反馈 | 玩家主观感受，不等于真实值 |
| 模糊提示文本 | 面向玩家的非数字反馈 |
| 信息可靠度 | 高 / 中 / 低 |
| 是否可查证 | 不可查证 / 可部分查证 / 可直接查证 |

玩家可用行为标签：

- 坦白。
- 追问。
- 回避。
- 撒谎。
- 冷处理。
- 查证。
- 道歉。
- 反击。
- 试探。
- 设边界。
- 逃避。
- 补偿。

提示文本示例：

- 你感觉对方不是完全没事，但也没有证据。
- 这句话听起来合理，只是细节有点少。
- 你越想越不舒服，但不确定是不是自己想太多。

## 4. 后果层

后果层记录事件结算，但不直接绑定固定数值。

| 字段 | 说明 |
| --- | --- |
| 真实数值变化 | 用方向描述，例如信任下降、压力上升 |
| 感知数值变化 | 玩家感知到的变化，可能与真实不同 |
| 破绽值变化 | 无变化 / 上升 / 明显上升 / 暂时降低 |
| 压力变化 | 双方压力分别变化 |
| 记忆记录 | 是否写入记忆账本，以及记录摘要 |
| 后续事件钩子 | 可能触发的追问、查证、旧账、第三者事件 |
| 可能触发的阶段变化 | 升温、冷淡、分手危机、继续暧昧等 |
| 修复机会 | 是否可以通过坦白、道歉、补偿、设边界修复 |

后果描述示例：

- 如果玩家选择“追问”，对高冲突回避 NPC 可能提高压力并引发回避。
- 如果 NPC 选择“半真半假”，短期冲突可能下降，但破绽值上升。
- 如果玩家选择“设边界”，对高责任感 NPC 可能提升信任，对高双标 NPC 可能引发反击。

## 6. 禁止写法

禁止：

- “选项 A：好感 +5”。
- “选项 B：信任 -10”。
- “NPC 一定原谅玩家”。
- “玩家追问一定发现真相”。
- “所有异性接触都等于第三者”。

推荐：

- “选项 A 使用行为标签：坦白。对高信任角色通常有利，但如果此前有未解决冲突，可能仍进入追问。”
- “选项 B 使用行为标签：冷处理。短期避免冲突，长期提高未解决冲突和旧账概率。”
