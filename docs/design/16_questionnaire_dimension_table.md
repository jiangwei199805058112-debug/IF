# 128维问卷维度表 v0.1

本文档定义 IF 完整调查问卷的底层维度表。它用于后续题库、评分、报告、NPC 档案、行为修正和事件权重设计。

本表不是医学或心理诊断工具，不用于判断现实人物好坏。所有维度只表示游戏内角色的概率性行为倾向。

## 1. 设计原则

1. **先定义维度，再写题库**：题库必须覆盖维度，不能先凭感觉写题。
2. **自述与行为分离**：问卷生成 `self_report_profile`，游戏行为继续生成 `behavior_profile`。
3. **高低值都不等于好坏**：高独立不一定冷漠，高亲密不一定健康，低怀疑也可能缺乏风险意识。
4. **避免单题定性**：每个维度至少需要多道题或游戏行为共同确认。
5. **服务模拟而非诊断**：维度用于事件概率、报告文本、NPC反应和长期记忆，不给现实玩家贴病理标签。
6. **允许矛盾人格**：同一角色可以同时高亲密需求、高回避亲密、高诚实自述、高行为隐瞒。

## 2. 规模定义

本版采用：

```text
16个大类 × 每类8个维度 = 128维
```

128维作为 IF 的完整深度框架。后续可以在附录中扩展到 160维，但第一版题库、报告和数据结构优先围绕 128维设计。

## 3. 字段说明

每个维度包含以下字段：

| 字段 | 含义 |
| --- | --- |
| 维度ID | 稳定英文ID，后续可作为配置字段或报告引用 |
| 中文名 | 玩家和设计文档可读名称 |
| 核心定义 | 该维度实际测量什么倾向 |
| 高值表现 | 高分角色更可能出现的行为或解释模式 |
| 低值表现 | 低分角色更可能出现的行为或解释模式 |
| 主要用途 | 影响的事件、报告或系统方向 |

建议后续实现使用 0-100 数值，但玩家报告只展示文字倾向，不直接暴露所有底层分数。

## 7. 题库覆盖要求

后续写 `18_super_realistic_question_bank.md` 时，题目应反向覆盖本表。

建议最低覆盖：

| 问卷模式 | 每个核心维度覆盖 | 说明 |
| --- | --- | --- |
| 快速版 | 只覆盖关键维度 | 25题，粗略画像 |
| 标准版 | 核心维度至少1题 | 45题，默认体验 |
| 深度版 | 核心维度1-2题 | 80题，较稳定 |
| 超级真实版 | 重要维度2-4题 | 100-180题，完整画像 |
| 研究/超长版 | 每维3-6题 | 需要250-500题，不作为第一版目标 |

注意：128维不代表第一版必须每维都精确测量。第一版可以做到：

- 重点维度高覆盖；
- 次要维度低覆盖；
- 依靠后续游戏行为继续修正；
- 报告中显示可信度和未充分确认的区域。

## 8. 评分与报告原则

### 8.1 评分来源

每个维度的分数可以来自：

1. 自评题；
2. 情境题；
3. 多选原因；
4. 主选项与次选项差异；
5. NPC视角题；
6. 反向验证题；
7. 游戏中真实行为。

### 8.2 报告表达

报告不直接说：

```text
你就是渣男。
你一定会出轨。
你有某种病态人格。
```

报告应写成：

```text
你在压力下更倾向于选择性表达。
你对自己和对方的异性社交标准不完全一致。
你在被质疑时容易先防御，再解释。
```

### 8.3 可信度

报告必须显示：

- 问卷完成度；
- 答案一致性；
- 自我美化风险；
- 双标风险；
- 哪些维度证据不足。

2. **身体与亲密节奏层**：非露骨地记录身体距离、亲密节奏、身体边界、照顾需求。
3. **文化与身份层**：跨文化恋爱、语言差异、身份公开、地区/家庭规范差异。
4. **长期成长与改变层**：学习能力、修复习惯、心理弹性、长期关系成长潜力。

160维不是当前目标。当前目标是先把128维作为稳定主表，用于下一步题型结构和题库设计。
