# 角色创建字段表 v0.1

本文件定义 IF 角色创建时需要记录的字段。v0.1 不追求字段完整实现，但要保证后续事件、问卷、数值和关系阶段有稳定输入。

## 1. 基础信息

| 字段 | 类型 | 示例值 | 影响系统 | v0.1 是否启用 |
| --- | --- | --- | --- | --- |
| 姓名 | 文本 | 林夏 | 事件文本、称呼、记忆账本 | 启用 |
| 性别 | 枚举 | 女 / 男 / 非二元 | 文本称谓、亲密边界、社会压力 | 启用 |
| 年龄/出生年月 | 数字或日期 | 24 / 2002-05 | 成年校验、人生阶段、家庭压力、亲密系统 | 启用 |
| 城市 | 文本或枚举 | 上海 | 见面成本、异地压力、日程 | 启用 |
| 当前关系阶段 | 枚举 | 暧昧中 | 事件池、可用选项、结局判断 | 启用 |
| 认识时长 | 数字加单位 | 3 个月 | 初始信任、共同记忆、边界默认值 | 启用 |

## 7. v0.1 字段使用原则

- 启用字段必须能影响至少一个种子事件。
- 部分启用字段可以先影响事件权重和反馈文本，不必进入完整公式。
- 暂不启用字段仍保留在表内，避免后续补系统时反复改角色结构。
- 成年校验是硬规则：涉及亲密、同居、开房、避孕、意外怀孕的机制只允许成年角色进入。
- 字段值不直接等于结果。事件结果必须结合关系阶段、当前压力、记忆账本和双方行为标签判断。
