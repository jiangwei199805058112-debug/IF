# 共有关系、交换关系与公平系统设计

本文档用于沉淀 IF 项目中的共有关系、交换关系、公平感、过度获益、获益不足、照顾动机和互惠规范机制。它对 `docs/design/30_social_exchange_dependency_system.md` 形成补充和修正：社会交换系统不能把亲密关系简单写成“贪婪地追求个人净收益”，因为成熟的亲密关系中还存在照顾对方幸福、共同体感、慷慨付出和非即时回报的规范。

核心原则：

```text
亲密关系里既有交换，也有共有。
交换关系强调付出与回报是否匹配；
共有关系强调对方的幸福是否也成为我的关心对象。
但共有不等于无限牺牲，公平也不等于每件事都五五分。
```

本设计参考亲密关系中的相互依赖、共有关系、交换关系、公平理论、过度获益、获益不足和心理公平等观点，并转译为 IF 的系统机制。本文档不收录原始截图或长篇原文，只保留可供项目使用的变量、事件模板、报告标签和落地边界。

---

## 1. 设计目的

`30_social_exchange_dependency_system.md` 已经定义了奖赏、代价、比较水平、替代选择、投入损失和依赖权力。该框架有用，但如果单独使用，可能把亲密关系误写成：

```text
每个人都只想用最小代价换最大收益。
```

这在陌生人、交易关系、普通合作中可能较适用，但在亲密关系中并不完整。很多伴侣会因为关心对方的幸福而自愿付出，即使短期内得不到等价回报。

因此本系统补充三个问题：

```text
1. 这段关系更像交换关系，还是共有关系？
2. 双方是否感到公平，而不是每件事是否完全相等？
3. 过度获益和获益不足如何影响内疚、怨恨、满意度和修复行为？
```

## 2. 系统一：交换关系与共有关系

### 2.1 设计原则

交换关系强调：

```text
我帮你，是因为期待你也回报我。
```

共有关系强调：

```text
我关心你的需要，即使没有立刻回报，我也愿意帮你。
```

亲密关系通常更接近共有关系，但并不意味着没有交换、没有公平、没有边界。更准确的口径是：

```text
亲密关系中的交换更长期、更模糊、更延迟，也更受信任和共同体感影响。
```

### 2.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `communal_orientation` | 共有倾向，是否主动关心伴侣需要 |
| `exchange_orientation` | 交换倾向，是否强调即时回报和对等付出 |
| `reciprocity_expectation` | 对回报的期待强度 |
| `delayed_reciprocity_tolerance` | 能否接受回报延迟 |
| `partner_welfare_concern` | 对伴侣幸福和需要的关心程度 |
| `need_responsiveness` | 对伴侣需要做出回应的动机 |
| `scorekeeping_tendency` | 记账倾向，是否精细计算谁付出更多 |
| `communal_strength` | 共有关系强度 |
| `exchange_shift_under_stress` | 不满时从共有转向交换记账的倾向 |

### 2.3 交换关系表现

| 场景 | 交换关系倾向 |
| --- | --- |
| 帮别人忙时 | 期待对方尽快回报 |
| 别人帮自己时 | 觉得自己欠债，需要还清 |
| 共同完成任务时 | 明确分清各自功劳 |
| 伴侣需要帮助时 | 先衡量自己能得到什么 |
| 自己不帮忙时 | 不太内疚，认为不是自己的责任 |
| 结婚或承诺时 | 可能更关注成本、收益和责任分割 |

### 2.4 共有关系表现

| 场景 | 共有关系倾向 |
| --- | --- |
| 帮别人忙时 | 不要求立刻回报 |
| 别人帮自己时 | 感到被关心，而不只是欠债 |
| 共同完成任务时 | 不明确区分谁的功劳 |
| 伴侣需要帮助时 | 更自然地回应对方需要 |
| 自己不帮忙时 | 可能产生内疚或关系责任感 |
| 结婚或承诺时 | 可能更满意，因为彼此幸福被纳入共同目标 |

## 3. 系统二：互惠不是立刻对等

### 3.1 设计原则

亲密关系里仍然存在互惠，但它通常不是即时的、清晰的、逐笔结算的。

更适合 IF 的规则是：

```text
短期可以不对等，长期需要可感知的公平和被珍惜。
```

如果双方都有较强共有倾向，一方短期付出更多并不一定降低满意度，前提是：

- 对方看见并珍惜这份付出；
- 未来存在回流或补偿；
- 付出者不是长期被剥削；
- 付出符合自愿，而不是被迫；
- 双方都把彼此幸福当成关系目标。

### 3.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `short_term_imbalance_tolerance` | 对短期不对等的容忍度 |
| `long_term_fairness_need` | 对长期公平的需求 |
| `felt_appreciation` | 付出是否被看见和珍惜 |
| `care_return_expectation` | 期待未来在需要时被照顾 |
| `unreciprocated_care_burden` | 付出长期无回应造成的负担 |
| `taken_for_granted_signal` | 被理所当然对待的信号 |
| `gratitude_expression` | 感激表达质量 |

## 4. 系统三：公平关系与比例公平

### 4.1 设计原则

公平不等于双方得到相同结果，也不等于双方付出完全一样。公平更接近：

```text
我的关系结果 / 我的关系贡献 ≈ 伴侣的关系结果 / 伴侣的关系贡献
```

这意味着：

- 一方贡献更多，得到更多结果，可能是公平的；
- 双方贡献相同，但一方得到更多，可能是不公平的；
- 一方贡献很少却得到很多，可能过度获益；
- 一方贡献很多却得到很少，可能获益不足。

### 4.2 核心变量

| 变量 ID | 含义 |
| --- | --- |
| `equity_ratio_self` | 自身结果/贡献比例 |
| `equity_ratio_partner` | 伴侣结果/贡献比例 |
| `perceived_equity` | 主观公平感 |
| `underbenefit_feeling` | 获益不足感 |
| `overbenefit_feeling` | 过度获益感 |
| `equity_distress` | 不公平带来的心理压力 |
| `fairness_restoration_drive` | 恢复公平的动机 |
| `psychological_equity_reframe` | 通过认知调整恢复心理公平 |

### 4.3 公平状态

| 状态 | 说明 | 可能情绪 |
| --- | --- | --- |
| 公平 | 双方结果与贡献比例接近 | 满意、稳定 |
| 获益不足 | 自己付出相对更多，得到相对更少 | 怨恨、委屈、愤怒、逃离倾向 |
| 过度获益 | 自己付出相对更少，得到相对更多 | 内疚、不安，也可能习以为常 |
| 心理公平化 | 通过解释、合理化或降低期待维持公平感 | 短期稳定，长期可能压抑 |

## 5. 系统四：过度获益与获益不足

### 5.1 设计原则

获益不足通常会更强烈地降低满意度，但过度获益也可能带来内疚、不安或自我合理化。IF 不能只模拟“吃亏的一方痛苦”，也要模拟“占便宜的一方是否有内疚、是否修复、是否变得理所当然”。

### 5.2 事件模板

事件：玩家长期负责大部分家务和情绪安抚，NPC 只偶尔表达感谢。

可能结果：

| 玩家解释 | 结果 |
| --- | --- |
| “对方最近压力大，我可以多承担一点。” | 短期可接受 |
| “对方一直看不见我的付出。” | 获益不足感上升 |
| “我不说，他/她永远不会主动。” | 怨恨和旧账上升 |
| “这就是我该做的。” | 心理公平化，但可能压抑需求 |

事件：NPC 发现玩家为自己做了很多，而自己很少回应。

可能结果：

| NPC 反应 | 结果 |
| --- | --- |
| 表达感激并主动补偿 | 公平感修复 |
| 感到内疚但不行动 | 内疚积压，关系尴尬 |
| 认为这是理所当然 | 玩家怨恨上升 |
| 反过来贬低玩家付出 | 高伤害，写入旧伤记忆 |

---

## 6. 系统五：公平恢复路径

当关系出现不公平感时，角色可能尝试恢复公平。

### 6.1 行为修复

| 方式 | 说明 |
| --- | --- |
| 增加自己的贡献 | 过度获益者主动承担更多 |
| 请求对方增加贡献 | 获益不足者提出具体需求 |
| 降低自己的贡献 | 获益不足者停止过度付出 |
| 提高对方结果 | 用补偿、照顾、感谢来回流 |
| 重新分工 | 明确责任，减少模糊消耗 |

### 6.2 心理修复

| 方式 | 风险 |
| --- | --- |
| “他/她比较特殊，所以应该被照顾” | 可能压抑自己的公平需求 |
| “我不需要那么多回报” | 可能短期稳定，长期委屈 |
| “这段关系本来就该我多付出” | 可能强化不平等 |
| “对方以后会补偿我” | 如果长期不发生，会转化为旧伤 |

### 6.3 退出或替代

如果公平长期无法恢复，角色可能：

- 减少投入；
- 寻找外部公平感；
- 发展婚外/关系外情感依靠；
- 提出分手；
- 留在关系中但进入冷淡或记账模式。

## 7. 共有关系中的公平特殊性

共有关系不是没有公平，而是公平的时间尺度更长、形式更模糊。

统一规则：

```text
交换关系关注即时对等；
共有关系关注长期被珍惜和需要时被回应。
```

在共有关系中，以下行为更重要：

- 伴侣是否看见我的付出；
- 伴侣是否在我需要时回应；
- 伴侣是否把我的幸福纳入考虑；
- 伴侣是否只在自己有需要时才靠近；
- 伴侣是否长期把我的付出视为理所当然。

## 8. 家务、照料与敏感公平领域

公平感在不同领域权重不同。对亲密关系尤其敏感的领域包括：

```text
家务劳动
照料小孩
情绪劳动
金钱压力
时间投入
社交配合
照顾双方父母
性亲密主动性
关系修复主动性
```

这些领域不公平时，伤害往往大于一般小事。

### 8.1 家务与照料事件模板

事件：同居后，玩家发现自己总是在收拾、提醒、安排生活细节。

系统判断：

- 实际贡献是否不均；
- 对方是否看见并感谢；
- 是否有重新分工机会；
- 玩家是否属于高公平敏感；
- 这是否触发旧家庭脚本；
- 对方是否把玩家的劳动视为理所当然。

## 9. 与 30、31、32 文档的关系

本系统对 `30_social_exchange_dependency_system.md` 做进一步细化：

- 30 处理奖赏、代价、CL、CLalt、投入、依赖和权力；
- 33 区分交换关系和共有关系，并处理公平感；
- 31 负责防止这些系统重复扣分；
- 32 负责区分接近奖赏和回避代价。

统一修正口径：

```text
30 的社会交换不是冷冰冰算计；
33 明确亲密关系中存在共有规范、长期互惠、慷慨照顾和公平敏感。
```

- 是否特别重视家务、照料和情绪劳动公平。

## 11. 报告标签设计

| 标签 ID | 含义 | 报告表达方向 |
| --- | --- | --- |
| `communal_oriented` | 共有倾向 | 更愿意把伴侣需要纳入自己的关心范围 |
| `exchange_oriented` | 交换倾向 | 更重视付出与回报是否清晰匹配 |
| `scorekeeping_pattern` | 记账模式 | 关系不满时容易开始计算谁付出更多 |
| `long_term_fairness_seeker` | 长期公平需求 | 能接受短期不对等，但需要长期被珍惜和回应 |
| `underbenefit_sensitive` | 获益不足敏感 | 付出多但回报少时容易累积委屈和怨恨 |
| `overbenefit_guilt` | 过度获益内疚 | 得到很多但付出较少时容易不安或想补偿 |
| `taken_for_granted_sensitive` | 被理所当然敏感 | 对伴侣忽视自己付出特别敏感 |
| `care_responsive` | 照顾回应型 | 愿意在伴侣需要时主动帮助和支持 |
| `equity_repair_capable` | 公平修复能力 | 能通过分工、表达感谢和调整贡献恢复公平 |

## 12. 事件库方向

后续可以设计以下事件：

1. 一方长期负责家务，另一方认为理所当然；
2. NPC 主动照顾玩家，但玩家没有回流；
3. 玩家开始在心里计算“我为你做了多少”；
4. 双方协商重新分配家务；
5. 一方短期低谷，另一方主动多承担；
6. 一方过度获益后表达感激并补偿；
7. 一方获益不足但选择心理公平化；
8. 一方要求“我帮你这么多，你也应该帮我”；
9. 共有关系在不满后转入交换记账模式；
10. 双方通过明确分工和感谢恢复公平。

---

## 13. 不做什么

本系统暂不做：

1. 不把亲密关系写成逐笔结算；
2. 不把慷慨付出写成愚蠢；
3. 不把共有关系写成无限牺牲；
4. 不把公平写成绝对五五分；
5. 不把获益不足者的不满写成单纯计较；
6. 不把过度获益者自动写成坏人；
7. 不把家务、照料、情绪劳动视为小事；
8. 不上传原始截图或大段原文；
9. 不接 AI API。

---

## 14. 一句话总结

```text
IF 的共有/交换/公平系统要模拟的不是“谁占便宜”，而是：
伴侣是否把彼此需要纳入共同生活，短期不对等是否被长期珍惜和回应所平衡，以及不公平感如何在家务、照料、情绪劳动和关系修复中累积或被修复。
```
