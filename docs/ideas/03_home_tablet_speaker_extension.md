# 家庭平板与音箱扩展创意

> 状态：未来画面/UI 与生活设备扩展创意。当前 `v0.1` 不开始正式 UI、美术、引擎迁移或音频系统实现。

## 1. 设计定位

家庭场景除了电脑、手机和桌面物品外，可以逐步升级出 **平板** 和 **音箱** 两类设备。

它们的作用不是替代手机和电脑，而是让家庭场景更有生活感，并提供额外的信息载体、氛围系统和轻量互动。

核心分工：

- 手机：聊天、电话、定位、即时通知、主要关系选择。
- 电脑：工作、金钱、网购、日程、长文本信息。
- 平板：家庭休闲、看视频、看相册、看日程、轻量网购、同居共享内容。
- 音箱：播放音乐、环境氛围、情绪调节、同居/约会气氛。

## 2. 平板系统

### 2.1 出现场景

平板主要出现在家庭场景，也可以出现在咖啡厅、床头柜或沙发场景。

可见位置：

- 桌面平放。
- 平板支架上立着。
- 沙发旁边。
- 床头柜。
- 同居后的客厅桌。

### 2.2 功能方向

平板可以承担以下功能：

- 看视频/综艺/电影。
- 查看相册。
- 查看日程。
- 轻量网购。
- 查看天气。
- 查看学习资料或电子书。
- 同居后共享购物清单、旅行计划、纪念日计划。
- 约会前查餐厅、路线、礼物。

### 2.3 与手机/电脑的区别

平板不做主要聊天入口，避免和手机重复。

建议定位：

- 手机是即时关系设备。
- 电脑是工作和财务设备。
- 平板是家庭休闲和共享计划设备。

### 2.4 物品等级

平板可分级，但不使用真实品牌名。

示例等级：

- 旧款平板：屏幕小，反应慢，价格低。
- 普通平板：满足日常娱乐和轻办公。
- 高级平板：屏幕更好，外观更精致。
- 旗舰平板：显示效果好，适合影音、绘画、计划整理。

性能影响建议：

- 低级平板主要影响使用舒适度。
- 高级平板提升娱乐体验、生活品质和画面美观度。
- 不应对亲密关系产生强制性高收益。

## 3. 音箱系统

### 3.1 出现场景

音箱主要出现在家庭桌面、客厅、卧室、同居场景，也可以出现在车内或咖啡厅作为环境音乐来源。

可见位置：

- 电脑旁。
- 书架上。
- 床头柜。
- 客厅桌。
- 电视柜。

### 3.2 功能方向

音箱可以播放音乐，作为家庭氛围系统。

可支持：

- 学习/工作背景音乐。
- 放松音乐。
- 约会氛围音乐。
- 失恋/低落时的情绪音乐。
- 同居时一起听歌。
- 对方来家里时的环境氛围。

### 3.3 音乐对系统的影响

音乐影响应较轻，不应变成强数值道具。

可影响：

- 心情小幅变化。
- 压力小幅下降。
- 学习/工作舒适度小幅提升。
- 家庭氛围感上升。
- 约会或同居场景的氛围表达。

限制：

- 音乐不能直接修复重大矛盾。
- 音乐不能替代解释、道歉、信任修复或长期互动。
- 音乐只作为环境修正和叙事氛围，不做核心关系捷径。

### 3.4 音箱等级

音箱可分级，不使用真实品牌名。

示例等级：

- 便宜小音箱：音质一般，占位小。
- 普通桌面音箱：日常够用。
- 高级桌面音箱：外观和音质更好。
- 旗舰音箱：价格高，氛围感强，适合同居和家庭空间升级。

## 4. 互动事件示例

平板事件：

- 对方来家里时，两人一起用平板选电影。
- 玩家用平板查礼物，被对象看到搜索记录。
- 同居后，平板上显示共享购物清单。
- 争吵后，玩家打开相册看到合影，触发轻微情感锚点。

音箱事件：

- 对方来家里，玩家播放对方喜欢的音乐。
- 玩家压力高时播放放松音乐，心情小幅恢复。
- 分手边缘时听到共同回忆的歌，触发记忆事件。
- 同居后，一方嫌音乐太吵，引发生活习惯冲突。

## 5. 代码层面未来需要预留

如果后续落地，需要在物品和场景系统中预留以下内容。

### 5.1 物品分类扩展

建议在 `InventoryItem / DeskItem` 的分类中增加：

```text
home_tablet
home_speaker
home_media_device
```

### 5.2 信息载体扩展

建议在 `DiegeticInfoSurface` 类型中增加：

```text
tablet_screen
speaker_status_light
media_panel
```

平板可以作为可点击放大的信息载体。音箱通常不需要复杂界面，可以通过按钮、状态灯、音乐文字提示或手机/平板控制。

### 5.3 状态结构扩展

平板状态建议字段：

```text
tablet_visible
tablet_location
tablet_zoomed
tablet_battery
tablet_mode
tablet_has_shared_plan
tablet_privacy_risk_tags
```

音箱状态建议字段：

```text
speaker_visible
speaker_location
speaker_on
current_music_mood
volume_level
music_source
noise_conflict_risk
```

### 5.4 服务扩展

未来可以增加：

- `HomeMediaDeviceService`：处理平板、音箱、音乐、影音、共享计划。
- `MusicMoodModifier`：处理音乐对心情、压力、氛围的轻微修正。
- `SharedHomePlanService`：处理同居后的购物清单、旅行计划、纪念日计划。

## 6. 当前非目标

当前不做：

- 真实音乐版权接入。
- 真实流媒体平台。
- 游戏内大规模音乐库。
- 平板完整操作系统。
- 音箱复杂音频模拟。
- 用平板替代手机作为主要聊天入口。

第一版最多只需要在设计上保留：家庭场景未来可以购买平板和音箱，并把它们作为桌面/家庭空间升级项。
