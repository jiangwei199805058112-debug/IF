# 生活场景画面创意池

> 状态：未来画面/UI 创意记录。当前 `v0.1` 仍以规则母版、纸面原型和控制台验证为主；本文不代表立即开始正式 UI、美术或引擎迁移。

## 1. 核心定位

IF 的未来画面不采用传统“独立属性框 + 独立行动栏”的界面，而采用更沉浸的 **生活场景内嵌信息界面**。

核心原则：

- 不照抄任何现有游戏的界面、资源、布局或图标。
- 可以参考生活模拟类游戏的轻量画风方向，但 IF 需要更写实一点。
- 行动点、金钱、今日任务、角色属性、天气、聊天入口等信息尽量融入实际物件。
- 玩家不需要频繁看到抽象 UI 框，而是通过书本、电脑、手机、车载屏幕等载体理解当前状态。
- 主要视角采用第一人称生活场景；玩家本人通常不显示。
- 聊天、约会、争吵等场景可以出现对方头像、半身立绘或模糊身影。

推荐风格：

- 场景：半写实现代生活插画。
- UI：清爽、现代、接近真实设备界面，但需要虚构化。
- 人物：轻半写实，不走夸张二次元方向。
- 参考方向：生活模拟的可读性 + 更现实的成年人/学生生活质感。

## 2. 身份驱动画面载体

不同人生阶段使用不同信息载体。

### 2.1 学生阶段

学生阶段不以电脑桌作为默认核心，而以课桌、书本、笔记本、手机作为主要载体。

主要画面：

- 教室课桌。
- 自习桌。
- 家里学习桌。
- 图书馆桌面。

信息载体：

- 书本/笔记本：显示今日课程、作业、考试、零花钱、体力、心情、天气、简单关系摘要。
- 手机：聊天、购物、定位、电话、通知、对象/朋友/家人消息。
- 书本文字可以模糊处理，只显示玩家需要读懂的摘要信息。

学生购物范围较小，主要包括：

- 水壶。
- 钢笔、文具。
- 早餐、零食、奶茶。
- 小礼物。
- 便宜衣物。
- 手办、车模、小摆件。
- 盆栽、仙人掌、小风扇等桌面装饰。

学生阶段的经济来源以零花钱、兼职收入、奖学金或家人支持为主。

### 2.2 成年人阶段

成年人阶段以电脑、手机、办公桌、家里电脑桌、车内场景为主要载体。

主要画面：

- 家里电脑桌。
- 公司办公桌。
- 咖啡厅桌面。
- 餐厅桌面。
- 车内驾驶座视角。
- 同居后的卧室/客厅/玄关桌面。

信息载体：

- 电脑：金钱、工资、今日工作、天气、日程、账单、房租、车贷、网购、电脑端聊天入口。
- 手机：聊天、电话、通知、定位、相册、支付、设置、隐藏通知、查手机风险。
- 车载屏幕：导航、来电、蓝牙、地点与驾驶状态。

成年人阶段的经济系统包括工资、奖金、副业收入、房租、车贷、保险、油费、停车费、礼物、约会、衣服、桌面物品和电子设备升级。

## 3. 手机系统

手机是关系推进和玩家选择的主要入口。

默认表现：

- 家里桌面：手机放在桌子右侧完整显示。
- 学生课桌：手机放在书本旁边，或压在书本下方一部分。
- 办公桌：手机放在电脑旁或桌子右侧。
- 车内：手机固定在手机支架上。
- 咖啡厅/餐厅：手机放在靠近玩家的一侧。

交互方式：

- 平时手机直接出现在场景里。
- 有消息时手机震动、亮屏或闪动。
- 点击手机后，手机平滑放大到屏幕中央，不做明显加载画面。
- 放大后进入虚构聊天软件、电话、定位、设置等界面。

聊天软件要求：

- 功能和使用习惯可以接近微信/聊天软件。
- 名称、图标、颜色、排版需要虚构化，避免直接使用真实产品商标和界面。
- 可包含聊天列表、联系人头像、置顶、未读、语音/电话、定位、相册、转账/礼物、隐藏通知。

第一版建议实现：

- 聊天列表。
- 对象、朋友、家人几个联系人。
- 点击头像进入聊天。
- 聊天里选择主要行为：真实回答、半真半假、撒谎、沉默、不回复、解释、道歉、发定位、要求对方发定位。

## 4. 电脑/书本信息界面

电脑和书本不只是装饰，而是当前生活状态的信息面板。

### 4.1 电脑

电脑主页面可以显示：

- 金钱余额。
- 今日工作。
- 当前时间。
- 天气。
- 日程。
- 邮件/公司消息。
- 网购入口。
- 账单、工资、生活成本。
- 电脑端聊天登录入口。

规则：

- 场景内电脑屏幕只显示大字摘要。
- 点击电脑后，电脑屏幕平滑放大，显示详细界面。
- 在家里或办公室可以登录电脑端聊天软件。
- 电脑端聊天更方便，但也会带来被别人看到屏幕的隐私风险。

### 4.2 书本

学生阶段的书本/笔记本可以显示：

- 今日课程。
- 作业。
- 考试。
- 零花钱。
- 体力/心情。
- 天气提醒。
- 简单关系提示。

规则：

- 书本上的装饰性文字可以模糊，不需要全部可读。
- 关键摘要必须清楚。
- 点击书本后进入详细学习/任务界面。

## 5. 桌面建造和物品系统

IF 后续可以加入桌面布置系统，但普通剧情模式和建造模式要分开。

普通模式：

- 点击手机、电脑、书本或物品互动。
- 不允许随意拖动核心物品，避免误操作和剧情演出冲突。

建造模式：

- 可拖动和摆放装饰品、外设、小物件。
- 退出后保存桌面布局。
- 物品位置应使用比例坐标保存，适配 1080p、2K、4K。

可移动/可摆放物品示例：

- 小花。
- 水杯。
- 鼠标。
- 鼠标垫。
- 耳机。
- 手办。
- 车模。
- 盆栽。
- 仙人掌。
- 小风扇。
- 对方送的礼物。
- 合影。
- 纪念物。

暂时固定的核心物品：

- 电脑屏幕。
- 手机基础位置。
- 书本基础位置。
- 主桌子。
- 大件家具。

## 6. 物品等级与命名

物品名称由 IF 自己设计，不直接使用现实品牌名。物品可以通过外观、价格、描述让玩家理解现实中的分级。

通用等级：

- 便宜款。
- 普通款。
- 高级款。
- 旗舰款。
- 限定款。
- 对方赠送款。
- 纪念款。

鼠标示例：

- 便宜鼠标：价格低，外观普通，耐用和手感较差。
- 普通鼠标：满足日常使用。
- 高级鼠标：外观、手感、稳定性更好。
- 旗舰鼠标：价格高，外观和质感明显提升。

性能差异建议：

- 可以有轻微性能区分，但不要做成强制数值碾压。
- 便宜物品可能轻微增加疲劳、降低工作/学习舒适度或美观度。
- 普通物品满足基本需求。
- 高级物品提供轻微效率、心情或生活品质收益。
- 旗舰物品主要体现审美、身份、满足感和小幅边际收益。
- 不要让玩家觉得不买旗舰就无法推进关系。

## 7. 关系物品与出轨/暧昧风险

对象相关物品可以降低出轨、暧昧和隐瞒风险，但降低幅度应较小。

设计理由：

- 真正有强烈出轨倾向、刺激追求或低承诺的人，不会因为桌上摆了合影就完全停止。
- 关系物品应起到提醒、牵制、情感锚点和安全感作用，而不是绝对锁死行为。

可作为关系锚点的物品：

- 合影。
- 情侣杯。
- 对方送的小花。
- 纪念日礼物。
- 对方送的钥匙扣。
- 一起买的摆件。
- 电脑桌面合照。
- 置顶聊天。

效果方向：

- 轻微降低暧昧冲动。
- 轻微降低隐瞒倾向。
- 提高争吵后的缓和概率。
- 提高对象安全感。
- 提高关系承诺提醒。
- 对象桌上摆放玩家相关物品时，也对对象产生类似影响。

限制：

- 风险降低权重必须小于人格、关系满意度、信任状态、未解决冲突、外部诱惑和长期互动模式。
- 关系物品不能完全阻止出轨或撒谎。

## 8. 桌面整洁与垃圾

桌面会随着时间、压力和生活习惯变乱。

可建模状态：

- 整洁度。
- 杂乱度。
- 垃圾数量。
- 被遮挡物品。

可能出现的垃圾/杂物：

- 外卖袋。
- 空瓶。
- 纸团。
- 没洗的杯子。
- 早餐包装。
- 乱放的书。
- 杂物堆。

影响：

- 操作不方便。
- 心情下降。
- 学习/工作效率下降。
- 对象来访时印象下降。
- 手机、书本或小物件可能被遮挡。

第一版可先做简单行为：点击垃圾即可清理，整洁度上升。

## 9. 天气系统

天气先使用游戏内随机天气，不接真实天气 API。

要求：

- 随机天气要符合月份、季节和地区大致规律。
- 雨季、夏季、冬季、换季应有不同天气权重。
- 天气可以显示在电脑、书本或手机主页。

天气可影响：

- 约会地点。
- 通勤迟到概率。
- 心情。
- 是否带伞。
- 是否开车接送。
- 关心型聊天事件。

示例关心话语：

- 今天下雨，记得带伞。
- 天气冷，多穿点。
- 今天很热，我给你买饮料吧。
- 雪大，我去接你。

## 10. 手机支架

手机支架是桌面/车内物品，不应显著改变核心画面可读性。

支架等级可以影响：

- 手机是否能立起来。
- 手机角度是否可调。
- 是否支持无线充电。
- 手机通知是否更容易看到。
- 车内是否能更合理显示导航和来电。

支架不应提供过强数值收益，主要提供便利、画面变化和生活质感。

## 11. 车内场景

车内采用第一人称驾驶座视角。

核心可见元素：

- 方向盘。
- 前挡风玻璃。
- 引擎盖边缘。
- 中控屏。
- 手机支架。
- 后视镜。
- 左侧车窗/后视镜边缘。

车辆可以升级，不同车型影响：

- 外观。
- 中控布局。
- 经济压力。
- 油费、保险、维修、停车费。
- 约会范围。
- 对象评价。
- 隐私风险，例如车载蓝牙暴露来电或消息。

车内选择主要通过点击手机放大完成。导航、蓝牙、车辆信息可以通过中控屏显示。

## 12. AI 图像与角色预设

当前阶段不接入 AI 图像生成，不接入联网生成服务，也不把本地生成模型集成到游戏运行时。

第一版采用预设头像、预设半身和预设模糊身影。

未来可以考虑：

- 开发外部资产生成流程。
- 使用 AI 工具离线生成头像/半身/场景素材。
- 人工筛选、修图、统一画风后再导入游戏。

不建议第一版做：

- 游戏内实时 AI 生成人物。
- 上传照片实时生成角色。
- 依赖玩家电脑本地跑图像模型。
- 使用真实人物脸进行仿真。

## 13. 画面分辨率

未来画面应支持 1080p、2K、4K。

建议：

- 逻辑设计分辨率：1920×1080。
- 背景与关键素材目标：3840×2160。
- 布局使用比例坐标、锚点、缩放和响应式 UI。
- 桌面物品摆放不直接保存绝对像素，而保存比例坐标、缩放、旋转和层级。

## 14. 当前非目标

当前 `v0.1` 仍不立即做以下内容：

- 正式 UI。
- 正式美术资产。
- Godot/Unity/Ren'Py 引擎迁移。
- 实时 AI 图像生成。
- 完整桌面建造系统。
- 完整 3D 车内驾驶场景。

本文只作为未来画面、生活场景和底层数据设计的创意池。
