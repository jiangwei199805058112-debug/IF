# 《IF》核心规则母版 v0.1

## 1. 游戏定位

IF 是一个从“已经认识”开始的现实亲密关系模拟器。它不以刷好感为核心，而是模拟两个人在时间、金钱、性格、家庭、工作、社交圈、亲密关系、隐瞒、撒谎和压力叠加下如何共同塑造关系走向。

系统重点：

- 不确定性：玩家无法确认对方真实想法，只能根据行为和反馈推断。
- 误判：同一行为可能来自忙碌、逃避、失望、隐瞒或试探。
- 双向主动：NPC 不是等待攻略的对象，也会选择、隐瞒、查证、冷战、分手或复合。
- 真实值与感知值偏差：系统内部记录真实关系状态，玩家只能看到模糊感知。
- 记忆后果：没有解决的问题会进入账本，在之后被翻出或叠加。

为什么存在：让亲密关系不再是单一路线和固定好感表，而是由双方性格、压力、边界和历史共同推动。

影响系统：开局、事件、关系阶段、对话反馈、NPC 行为、结局判断。

## 2. 开局模式

最低开局不是陌生人。允许的开局只有：

- 已认识但不熟。
- 正在聊天。
- 暧昧中。
- 刚确认恋爱。
- 恋爱一段时间。
- 分手后复联。

每种开局都要有基础历史：

- 认识渠道。
- 认识时长。
- 当前聊天频率。
- 是否有共同朋友。
- 是否见过面。
- 是否已有未解决误会。
- 是否存在前任、异性朋友或家庭压力。

为什么存在：现实亲密关系的张力来自既有印象和历史，不适合从完全陌生人零接触开始。

影响系统：初始信任、安全感、边界协议、社交圈交集、事件触发池。

## 3. 动态关系阶段

关系不按固定月份推进，而按数值、事件、记忆和选择动态推进。阶段包括：

- 聊天。
- 暧昧。
- 确认关系。
- 热恋。
- 磨合。
- 冷淡。
- 分手危机。
- 分手。
- 复合。
- 同居。
- 求婚。
- 长期稳定。
- 分分合合。

阶段判断规则：

- 阶段由多维综合决定，不由单一好感决定。
- 关键事件可以直接推动阶段变化，例如重大隐瞒暴露、公开关系、见家长、同居提议。
- 原则性错误可以绕过慢速累积，直接进入分手危机。
- 同一阶段可以反复进入，例如热恋后冷淡，冷淡后复合，复合后再次分手危机。

为什么存在：关系发展不是线性升级，而是会反复、卡住、倒退或突然跃迁。

影响系统：事件池、NPC 主动行为、玩家可选行动、结局倾向。

## 4. 关系可行性初筛

开局前先做可行性初筛，用于判断两人的关系是否有基础推进空间。初筛不是最终结局，只决定初始张力和风险。

核心维度：

- 年龄与人生阶段是否兼容。
- 城市距离与见面成本。
- 工作/上学/求职时间是否可协调。
- 金钱观与消费压力。
- 公开关系意愿。
- 亲密节奏边界。
- 前任和异性朋友边界。
- 家庭压力。
- 情绪处理方式。
- 责任感与未来规划。

筛选结果：

- 高可行：矛盾较少，但仍会被事件和压力改变。
- 中可行：可以推进，但需要处理明显冲突。
- 低可行：有强张力和高风险，容易进入拉扯、冷淡或分手危机。
- 暂不可行：当前条件不支持稳定推进，但可以作为分手复联、异地拉扯或短期暧昧测试素材。

为什么存在：避免所有角色都能无条件发展，也让开局差异影响后续行为。

影响系统：初始阶段、冲突概率、事件权重、关系稳定度。

## 5. 时间与日程系统

v0.1 以天为主循环，每天分为早上、白天、晚上、深夜四个时间段。

角色基础日程：

- 上班。
- 上学。
- 求职。
- 休息。
- 加班/考试/面试。
- 家庭事务。
- 社交安排。

日程影响：

- 可回复消息时间。
- 见面机会。
- 情绪余量。
- 消费能力。
- 熬夜概率。
- 撒谎或隐瞒的诱因。

微信没有已读机制。玩家不能直接知道“对方看没看”，只能根据回复时间、朋友圈动态、共同朋友和后续解释判断。

为什么存在：亲密关系中的误会常来自时间错位，而不是单纯喜欢或不喜欢。

影响系统：消息事件、约会事件、压力、冷战、破绽。

## 6. 真实值与感知值

系统内部记录真实值，玩家看不到对方真实好感、真实信任、真实安全感、真实分手念头。

真实值示例：

- 真实好感。
- 真实信任。
- 真实安全感。
- 真实压力。
- 真实矛盾。
- 真实失望。
- 生理吸引。
- 心理喜欢。
- 分手念头。
- 愧疚感。
- 破绽值。
- 关系满意度。

玩家只看到感知反馈，例如：

- 对方似乎冷淡了一点。
- 你感觉她还在意你，但不太愿意说。
- 他最近回复变慢，你不确定是忙还是在逃避。
- 这次解释让你感觉不太对。

感知值来源：

- 可观察行为。
- 对方表达。
- 玩家性格和不安程度。
- 共同朋友反馈。
- 过往记忆。
- 当前压力。

为什么存在：玩家的判断应当像现实关系一样不完整，而不是看面板做最优解。

影响系统：反馈文本、玩家误判、查证行为、冲突升级。

## 7. 双向行为原则

玩家能做的行为，NPC 理论上也能做。

NPC 不是被动恋爱对象，而是有自己的时间、压力、性格、欲望、隐瞒、选择和分手权。

NPC 可以：

- 撒谎。
- 半真半假。
- 隐瞒异性饭局。
- 不回消息。
- 冷战。
- 拉黑。
- 查证玩家。
- 试探。
- 暧昧别人。
- 骑驴找马。
- 脚踏两只船。
- 使用双重账号/两部手机。
- 自己能做但不允许玩家做。

NPC 主动行为由以下因素共同决定：

- 性格维度。
- 当前压力。
- 对关系的满意度。
- 对玩家的信任与失望。
- 社交机会。
- 边界协议。
- 过去行为是否成功。
- 被发现风险。

为什么存在：关系模拟必须是双方博弈和共同塑造，而不是玩家单方面操作。

影响系统：事件触发、隐藏行为、第三者、分手权、复合可能。

## 8. 撒谎、隐瞒与破绽系统

撒谎不是纯随机。它由角色性格、情境收益、风险和关系状态共同决定。

规则描述：

```text
撒谎倾向 = 诚实倾向 + 冲突回避 + 当前压力 + 撒谎收益 + 被发现风险 + 过往成功经验 + 愧疚感 + 关系满意度
```

说明：

- 诚实倾向在计算中作为反向约束，诚实越高，越倾向坦白或半真半假。
- 冲突回避越高，越可能模糊回答、隐瞒重点或冷处理。
- 当前压力越高，越容易选择短期省事的说法。
- 撒谎收益越高，越可能隐瞒。
- 被发现风险越高，可能降低撒谎，也可能促使更复杂的谎言。
- 过往成功经验越多，越容易重复同类行为。
- 愧疚感越高，可能促成坦白，也可能导致防御性反击。
- 关系满意度越低，越可能降低坦白意愿。

撒谎类型：

- 完全坦白。
- 半真半假。
- 隐瞒重点。
- 模糊回答。
- 完全撒谎。
- 反向指责。
- 冷处理。
- 善意谎言。
- 逃避谎言。

隐瞒与撒谎的区别：

- 撒谎：主动给出与真实情况不符的信息。
- 隐瞒：不主动提及重点事实。
- 半真半假：事实局部真实，但省略关键对象、目的或时间。

为什么存在：现实冲突中更常见的是半真半假和隐瞒重点，而不是简单真话/假话。

影响系统：信任、破绽值、记忆账本、第三者、分手危机。

## 9. 忠诚、第三者与双重账号系统

外部异性对象可以出现，但不必每次都发展成背叛。系统要区分边界、试探、隐瞒和实质关系变化。

双方可以选择：

- 保持边界。
- 普通接触。
- 暧昧试探。
- 隐瞒接触。
- 坦白。
- 分手后发展。
- 骑驴找马。
- 脚踏两只船。

第三者事件来源：

- 同事、同学、朋友、前任、网友。
- 饭局、电话、朋友圈互动、深夜聊天、共同朋友提及。
- 对方低谷时的情绪支持。
- 玩家长期忽视或冲突后出现的替代关系。

双重账号/两部手机规则：

- 不是所有角色都有该行为，只由高隐瞒倾向、高社交机会、高被发现成本等条件触发。
- 可以造成线索缺口，例如某些时间段解释不完整。
- 暴露后通常是高强度信任危机。

为什么存在：忠诚不是静态标签，而是持续边界选择。

影响系统：信任、安全感、破绽、分手念头、复合难度。

## 10. 性格、双标与边界协议

性格不能写成单一标签，要拆成可计算维度。

建议维度：

- 依恋安全感。
- 冲突回避。
- 表达直接度。
- 占有欲。
- 共情能力。
- 责任感。
- 金钱敏感度。
- 社交展示欲。
- 撒谎倾向。
- 双重标准倾向。
- 边界敏感度。
- 情绪恢复速度。

双标规则：

- 角色可以允许自己做某事，却不允许对方做同样的事。
- 双标不必直接等于恶意，它可能来自不安全感、控制欲、家庭经验、过去背叛或自我合理化。
- 双标被指出后，角色可能承认、辩解、反击、转移话题或冷处理。

边界协议：

- 是否可以单独和异性吃饭。
- 是否公开关系。
- 是否查看手机。
- 是否联系前任。
- 是否接受深夜聊天。
- 是否接受临时失联。
- 亲密节奏。
- 金钱和礼物边界。

为什么存在：关系冲突常发生在双方默认边界不同，而不是规则已经说清。

影响系统：问卷、事件判断、冲突生成、记忆旧账。

## 11. 亲密、同居、避孕与意外怀孕系统

本系统只写规则，不写露骨内容。

所有亲密相关机制只适用于成年人。未成年角色不进入该系统。

亲密系统维度：

- 生理吸引。
- 心理喜欢。
- 亲密接受节奏。
- 安全感。
- 责任感。
- 边界表达能力。
- 当前关系阶段。
- 当前压力和矛盾。

规则：

- 开房/过夜默认进入亲密判定，但不等于一定发生亲密行为。
- 双方意愿、边界、关系阶段和压力共同决定结果。
- 避孕选择只做系统规则和非露骨文字。
- 意外怀孕风险作为低概率高影响事件，只用于成年人角色。
- 意外怀孕会引入责任感、家庭压力、经济压力、求婚/分手/同居压力。
- 亲密后可能提升亲近感，也可能因为节奏不一致、失望或压力造成关系波动。

为什么存在：现实亲密关系会影响安全感、责任、未来规划和家庭压力，不能只作为好感奖励。

影响系统：安全感、责任感、家庭压力、经济压力、阶段变化、结局。

## 12. 经济、餐厅、礼物与共同开销

经济系统关注金钱观差异，不只记录余额。

核心维度：

- 个人收入。
- 家庭收入。
- 消费习惯。
- 金钱敏感度。
- 礼物期待。
- 餐厅消费接受度。
- AA/请客/轮流规则。
- 共同开销承担方式。
- 物质需求和社交展示欲。

事件类型：

- 节日礼物。
- 约会餐厅选择。
- 临时借钱。
- 共同旅行。
- 同居开销。
- 对方将消费细节发朋友圈。

为什么存在：金钱不是单纯资源，而是安全感、被重视感、价值观和家庭背景的冲突点。

影响系统：失望、矛盾、安全感、关系满意度、未来规划。

## 13. 冷战、熬夜、拉黑与未解决冲突

冲突不一定当天解决。没有解决的问题会进入未解决冲突列表。

行为类型：

- 冷战。
- 延迟回复。
- 拉黑。
- 删除好友。
- 熬夜争吵。
- 第二天装作没事。
- 道歉但不解决核心问题。
- 表面和好。

后果：

- 短期降低冲突表达。
- 提升未解决冲突。
- 降低安全感。
- 提升失望。
- 影响第二天精神状态。
- 增加旧账触发概率。

为什么存在：很多关系不是一次冲突结束，而是未解决问题反复堆积。

影响系统：记忆账本、压力、健康状态、消息事件、分手危机。

## 14. 公共场合吵架与私下解决

吵架地点会影响后果。

公共场合冲突：

- 容易引入羞耻感、面子压力、朋友评价和旁观者干扰。
- 高表达直接度角色可能当场爆发。
- 高社交展示欲或高面子压力角色更容易记仇。

私下解决：

- 更适合深入沟通。
- 也可能变成冷处理或无限拖延。
- 如果一方多次要求私下解决但之后不解决，会进入记忆账本。

玩家选择：

- 当场说清。
- 忍住回家说。
- 先离开现场。
- 转移话题。
- 公开反击。
- 私下道歉。

为什么存在：同一矛盾在不同场合爆发，后果完全不同。

影响系统：面子压力、失望、朋友评价、冲突升级、记忆。

## 15. 压力叠加系统

工作、学校、求职、家庭、经济、感情、社交、身体、亲密压力可以同时出现。

压力来源：

- 工作加班。
- 考试或论文。
- 求职失败。
- 家庭催促。
- 经济紧张。
- 朋友评价。
- 前任出现。
- 亲密节奏不一致。
- 睡眠不足。

角色过载后会：

- 回消息变慢。
- 判断力下降。
- 更容易撒谎。
- 更容易冷战。
- 更容易熬夜。
- 更容易爆发。
- 更难照顾对方情绪。

压力不等于不爱。系统必须允许“还喜欢，但没有余力处理关系”的状态。

为什么存在：现实中的冷淡经常来自压力和精力不足，而不是单一感情下降。

影响系统：日程、消息、撒谎、冲突、亲密、安全感。

## 16. 记忆账本与旧账系统

记忆账本记录重要事件，不记录所有细节。

进入账本的内容：

- 原则性错误。
- 未解决冲突。
- 重大撒谎或隐瞒。
- 公共场合羞辱。
- 重要节日落空。
- 亲密边界被忽视。
- 对方曾经主动修复关系。
- 高价值支持行为。

记忆字段：

- 事件摘要。
- 当时真实情况。
- 当时玩家感知。
- 是否解决。
- 当前情绪残留。
- 可被旧账触发的条件。
- 对信任、安全感、失望的长期影响。

旧账触发：

- 相似事件再次发生。
- 熬夜争吵。
- 第三者线索出现。
- 节日或纪念日。
- 分手危机。

为什么存在：关系历史会改变同一句话、同一行为的含义。

影响系统：事件解释、冲突强度、复合难度、结局。

## 17. v0.1 最小测试范围

v0.1 不做一年，而做：

```text
14天暧昧/刚恋爱高密度测试版
```

范围：

- 开局：正在聊天 / 暧昧中 / 刚恋爱。
- 时间：14 天。
- 日程：上班 / 上学 / 求职。
- 事件：先 3 个种子事件，之后扩到 20 个。
- 数值：好感、信任、安全感、压力、矛盾、失望、生理吸引、心理喜欢。
- 隐藏机制：真实值/感知值、撒谎、破绽值、未解决冲突。
- 结局：升温、确定关系、继续暧昧、冷淡、分手危机、分手、分分合合倾向。

v0.1 验证目标：

- 玩家是否能在信息不完整时做选择。
- NPC 主动行为是否能推动关系变化。
- 三层事件卡是否足够表达真实、感知和后果差异。
- 压力是否能解释非恶意冷淡。
- 未解决冲突是否能自然变成旧账。

## 18. 当前不做内容

当前阶段不做：

- Python/JS/Unity/Godot 游戏代码。
- UI 或视觉资产。
- AI API 接入。
- 事件引擎实现。
- 数据库。
- 100 个事件。
- 完整一年模拟。
- 露骨性描写。
- 完全陌生人相识模拟。
- 把数值直接写死成“选项 A 好感 +5”的结构。

当前阶段只做规则母版、字段设计、问卷设计、数值系统、事件模板和少量种子事件。
