# 14 天流程缺少游戏感待办

本文档记录一次真实试玩后的核心体验反馈：当前 14 天互动流程更像关系规则测试器，而不像可玩的游戏。

玩家反馈：

```text
现在没有玩明白那个 14 天的东西。
它好像仅仅就是给我出了一个问题，让我做出回答，完全没有游戏的感觉。
```

判断：反馈成立。当前 14 天流程的主要功能是验证问卷、关系事件、aggregator、RelationshipMemory、NPC 反应和报告链路，不是完整玩法层。

---

## 1. 当前问题

当前 14 天流程大致是：

```text
固定日程文本
→ 到事件日弹出一个问题
→ 玩家选择一个处理方式
→ 输出关系变化摘要
→ 最后输出阶段结算
```

这对规则验证有效，但对玩家来说缺少：

```text
日常行动选择
聊天推进
NPC 主动回应
地点 / 时间感
关系目标
短期任务
连续反馈
可理解的状态变化
玩家主动感
```

所以玩家会感觉：

```text
我只是被问了几个题。
我不知道自己在玩什么。
我不知道每天能做什么。
我不知道角色为什么这样反应。
```

---

## 2. 根本原因

### 2.1 当前流程是测试框架，不是游戏循环

14 天流程最初用于测试：

```text
消息延迟
异性饭局 / 边界
吵架后修复
冷处理
关系报告
记忆写入
```

因此它的结构是“事件验证优先”，不是“玩家体验优先”。

### 2.2 缺少每日可玩动作

目前玩家在非事件日基本只能看提示，不能真正选择：

```text
发消息
约见面
送东西
学习 / 工作
观察对方朋友圈
主动解释
冷处理
找朋友倾诉
自我恢复
推进关系
保持距离
```

### 2.3 缺少 NPC 主动性

NPC 现在大多是事件里的固定解释，而不是每天根据人格和关系状态主动发起：

```text
主动找你聊天
突然冷淡
试探你
道歉
提出见面
表达不满
转移话题
设置边界
```

### 2.4 缺少短期目标

玩家不知道 14 天内要达成什么，例如：

```text
把关系推进到暧昧
稳定聊天节奏
处理一次边界冲突
修复一次误会
避免关系进入冷淡
确认是否继续发展
```

---

## 3. 后续目标：从规则测试器变成可试玩 MVP

建议下一阶段不要继续扩底层关系系统，而是补一个“游戏感壳层”。

建议版本：

```text
v0.1.58 14 天互动玩法壳层 MVP
```

如果 `current_task_ledger.md` 已有新版本号安排，以账本为准。

目标不是做完整 UI，也不是做 AI 对话，而是在控制台中增加基础游戏循环：

```text
每天开始
→ 显示日期 / 阶段 / 当前氛围
→ 给玩家 3-5 个日常行动
→ NPC 根据人格和状态做简短回应
→ 关系状态 / 记忆 / 事件概率轻量变化
→ 特殊事件日再进入关键事件选择
→ 每 3-4 天给阶段反馈
→ 第 14 天结算
```

## 4. 建议玩法结构

### 4.1 每日开头

示例：

```text
第 4 天｜正在聊天｜氛围：有点不确定
昨晚你们的聊天停在一句“我先忙”。
今天你想怎么做？
```

### 4.2 每日行动菜单

建议每个普通日给 4 个选项：

```text
1. 主动发一条轻松消息
2. 认真问一下对方最近忙不忙
3. 等对方主动联系
4. 先做自己的事，减少查看消息
```

不同阶段可替换为：

```text
约见面
分享日常
表达关心
开玩笑试探
认真推进关系
谈边界
暂时保持距离
```

### 4.3 NPC 回应

NPC 回应不需要 AI，只需模板 + 人格分支，例如：

```text
忙碌回避型：回复短，但不是完全消失。
高吃醋双标型：会追问你的社交，但自己解释不多。
成熟沟通型：更容易解释原因并提出下次怎么做。
情绪试探型：可能用冷淡或反问测试你是否在意。
```

### 4.4 玩家可见状态

不显示真实数值，但显示模糊状态：

```text
当前氛围：稳定 / 微妙 / 降温 / 紧张 / 危机
对方主动性：正常 / 变少 / 明显回避
你的不安：低 / 中 / 高
修复窗口：开放 / 变窄 / 暂时关闭
```

### 4.5 每几天小结

例如第 3、7、10、14 天输出：

```text
这几天你们联系不少，但有些话没有说透。
对方没有明显疏远，不过边界问题还没有真正解决。
```

## 5. 建议先做的最小实现

不要一次做完整游戏。建议先做最小可玩壳层：

```text
1. 增加 daily_action_choices。
2. 普通日也让玩家做一次选择。
3. NPC 根据 profile_pair / personality 给一句回应。
4. 普通日行动产生轻量 relationship_delta。
5. transcript 中显示“你做了什么 / 对方怎么回应 / 氛围变化”。
6. 保留现有第 3、8、12 天关键事件。
```

这样 14 天就不再是只有 3 道题，而是每天都有轻量参与。

## 8. 验收标准

完成后，玩家再运行：

```powershell
python -m if_game.main
```

应该不再只是：

```text
第 3 天出一道题
第 8 天出一道题
第 12 天出一道题
```

而应该更像：

```text
第 1 天：确认关系节奏，选择今天怎么聊天。
第 2 天：普通日，你选择主动分享日常，对方有回应。
第 3 天：消息延迟事件，进入关键选择。
第 4 天：余波日，你选择要不要继续提起。
...
第 14 天：阶段结算。
```

---

## 9. 当前结论

当前 14 天流程不是失败，而是它完成的是“规则验证”，还没完成“玩法层”。

下一步应优先补：

```text
每日行动
NPC 回应
氛围反馈
阶段目标
```

这样才会从测试器变成游戏原型。
