# v0.1.58 真实试玩反馈与下一步修复待办

本文档记录 v0.1.58 `14 天互动玩法壳层 MVP` 完成后的真实试玩反馈。

试玩 commit：

```text
c75d5b79847f5518295e075671e665055fcea459
feat: 增加14天互动玩法壳层
```

玩家核心反馈：

```text
做完每个选项后，看不到对方的反馈反应。
现在也不太懂是在做底层，还是在做什么。
如果真实游戏是这样，应该还是很无聊。
还是需要有自己的生活、自己的选项，然后和对方的互动。
```

结论：反馈成立。v0.1.58 已经把普通日行动、NPC 回应、氛围头部、余波日和中期反馈写入最终 transcript，但交互过程仍然是“连续选择 → 最后一次性输出模拟记录”。玩家在选择当下无法即时看到 NPC 回应，因此实际体验仍然像在填测试表。

---

## 1. 当前问题

### 1.1 反馈延迟

当前交互阶段大致是：

```text
第 1 天选择行动
第 2 天选择行动
第 3 天选择事件处理
...
第 13 天选择行动
→ 最后统一打印 14 天 transcript
```

问题是：

```text
玩家每次选择后，看不到立即反馈。
NPC 回应虽然写进 transcript，但被延迟到最后才展示。
玩家无法形成“我刚刚做了什么 → 对方马上怎么回 → 氛围怎么变”的闭环。
```

因此 v0.1.58 从结构上已经接近游戏，但操作体验仍像问卷 / 批量模拟器。

---

### 1.2 生活玩法仍缺失

当前 14 天流程仍以恋爱互动为唯一主线，缺少玩家自己的日常生活。

真实游戏需要同时存在：

```text
玩家自己的生活
玩家自己的时间和精力
学习 / 工作 / 兼职 / 休息
朋友 / 社交 / 自我状态
与 NPC 的聊天、见面、冲突和修复
```

否则玩家会觉得：

```text
我每天只是围着对方做选择。
我没有自己的生活。
关系像测试题，不像生活模拟。
```

---

### 1.3 组合选择仍不清楚

试玩中还发现：

```text
“选择样例角色组合”不清楚是在选自己、选对方，还是选关系剧本。
当前 A/B/C/D/E 是测试预设组合，不应被表达成正式游戏的唯一关系类型。
```

后续应改为：

```text
快速预设组合
自定义关系组合
根据问卷生成
```

其中自定义关系组合应拆分：

```text
玩家倾向
NPC 倾向
关系阶段
主要矛盾主题
```

## 2. 下一步优先级判断

当前最高优先级不是继续扩底层，也不是继续扩问卷，而是修复“即时反馈缺失”。

建议下一版本：

```text
v0.1.59 即时反馈与逐日交互修复
```

核心目标：

```text
玩家每完成一次选择，就立刻看到当天的状态头部、自己的行动、NPC 回应、氛围变化和必要记忆反馈。
```

这一步完成后，再继续做：

```text
v0.1.60 开局关系选择拆分：快速预设 / 问卷生成 / 手动自定义
v0.1.61 玩家生活行动层：学习 / 工作 / 休息 / 社交 / 恋爱互动的时间分配
```

如 current_task_ledger.md 已有不同版本安排，以账本为准，但优先级建议保持上述顺序。

## 3. v0.1.59 建议目标：即时反馈与逐日交互

### 3.1 必须改变的体验

从：

```text
连续做完所有选择
→ 最后统一输出模拟记录
```

改为：

```text
每天开始
→ 显示当天状态头部
→ 玩家选择行动 / 处理事件
→ 立即显示：
   - 你的行动
   - NPC 回应
   - 氛围变化
   - 你的感受
   - 必要的关系变化摘要
→ 按回车进入下一天
```

---

### 3.2 示例体验

```text
第 4 天｜正在聊天
氛围：紧张
对方主动性：明显回避
你的不安：高
修复窗口：暂时关闭

昨天你把不满说得比较直接，对方今天的回复明显谨慎了一点。

今天你想怎么做？
1. 主动发一条轻松消息
2. 认真问对方最近是不是很忙
3. 分享一点自己的日常
4. 等对方主动联系
5. 先做自己的事，减少查看消息

请输入：1

你选择：主动发一条轻松消息
许知遥很快回：“你今天好像有点不一样，是不是在试我？”

氛围变化：气氛变得有点微妙，主动性没有完全接上。
你的感受：你把今天的互动当作一个小信号，而不是最终结论。

按回车进入第 5 天。
```

## 4. v0.1.59 实现建议

### 4.1 engine.py 调整方向

当前 `run_14_day_simulation(interactive=True)` 应避免只收集输入、最后输出 transcript。

建议改为：

```text
interactive=True 时：
- 每天即时 print 当天输出。
- 当天输出仍同步写入 transcript。
- 最后第 14 天可以保留完整记录或只输出阶段总结。

interactive=False / scenario 测试时：
- 继续只返回 transcript，不做输入等待。
```

关键原则：

```text
互动模式必须即时反馈。
非互动测试模式必须保持可测试、可断言。
```

---

### 4.2 输出控制

可新增 helper：

```python
def emit_interactive_lines(lines: list[str], interactive: bool) -> None:
    if interactive:
        for line in lines:
            print(line)
```

或类似结构，避免把 print 散落到所有逻辑中。

要求：

```text
同一份 lines 既进入 transcript，又能在 interactive 模式即时显示。
```

---

### 4.3 每天之间加节奏控制

建议互动模式每天结束后提示：

```text
按回车进入下一天。
```

但测试模式不要等待输入。

---

### 4.4 关键事件也要即时反馈

第 3 / 8 / 12 天关键事件选择后，也要立即显示：

```text
你的选择
记忆账本
关系状态变化
感知反馈
```

不要等到第 14 天统一输出。

## 6. v0.1.61 建议目标：玩家生活行动层

该问题来自同次试玩。

玩家反馈：

```text
如果真实游戏是这样，应该很无聊。
还是需要有自己的生活、自己的选项，然后和对方的互动。
```

判断：反馈成立。

后续应增加“玩家自己的生活”，让每日选择不只是围着 NPC 转。

建议每日行动拆成两层：

```text
今天主要怎么安排？
1. 学习 / 工作
2. 兼职 / 赚钱
3. 休息 / 恢复状态
4. 朋友 / 社交
5. 恋爱互动
```

如果选择恋爱互动，再进入当前 v0.1.58 的恋爱行动菜单：

```text
1. 主动发轻松消息
2. 认真问对方近况
3. 分享日常
4. 等对方主动
5. 减少查看消息
```

这样能形成：

```text
生活资源 → 恋爱投入 → 关系反馈
```

而不是每天只做恋爱题。

## 7. 不建议现在一次性做的内容

v0.1.59 不建议直接做：

```text
完整生活系统
完整经济系统
自由地图
多 NPC
AI 对话
复杂时间管理
大型剧情树
```

原因：

```text
当前最短板是即时反馈。
如果不先修复即时反馈，即使增加生活系统，玩家仍然会觉得像批量答题。
```

因此顺序应为：

```text
v0.1.59 先修即时反馈
v0.1.60 再修开局选择表达
v0.1.61 再补玩家生活行动层 MVP
```
