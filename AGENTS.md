# AGENTS.md

本文件给后续 Codex/AI 开发助手使用。所有后续工作必须先遵守这里的项目边界，再执行具体任务。

## 项目边界

- 第一阶段只做设计文档和纸面原型。
- 规则未闭环前不写正式游戏代码。
- 不做完整一年模拟，v0.1 只验证 14 天暧昧/刚恋爱高密度测试版。
- 不做画面/UI。
- 不做完全陌生人从零认识。
- 不做露骨性内容。
- 所有亲密、开房、同居、避孕、意外怀孕内容只做系统规则和非露骨文字。
- 所有角色涉及亲密关系时默认必须是成年人。
- 未成年/高中生不进入性行为、同居、开房、怀孕系统。

## 开发规则

- 禁止 force push。
- 禁止 `git reset --hard`，除非用户明确要求。
- 提交前检查 `git status --short` 和 `git diff --check`。
- 每次修改要小步提交，提交信息要说明具体设计范围。
- 不要用 `git add .` 盲目提交，优先明确添加文件。
- 文档更新可以不跑代码测试。
- 后续如果写代码，必须有最小 smoke test。
- 不删除用户已有文件，除非明确确认是本助手刚创建且错误的文件。

## 设计规则

- 玩家和 NPC 都可以主动行为。
- 玩家看不到真实数值，只能看到感知反馈。
- 事件必须支持真实层、感知层、后果层。
- 撒谎、隐瞒、双标、第三者、压力叠加、冷战、拉黑、熬夜、记忆系统都是核心机制。
- 不要把性格做成单一标签，要拆成可计算维度。
- 不要把数值直接写成“选项 A 好感 +5”。事件应使用行为标签、上下文和角色参数计算结果。
- 高风险行为不做道德说教，但必须有合理后果，并进入记忆、信任、压力或阶段系统。
